# 한글 수식입력기 코드 규칙 (hmr)

이 skill은 **수식을 어떻게 코드로 적을까**를 다룬다. 풀이의 의미·용어·서술은 `hsr` skill.

## 단일 정본 (SOT)

📄 **`한글 수식입력기_260425.txt`** (프로젝트 루트)

본 skill은 그 정본의 트리거·핵심 요약·자주 보는 빠른참조표일 뿐이다.
편집·확장은 항상 SOT 파일 본체에서 한다 (역방향 금지).

## 언제 호출해야 하는가

| 상황 | hmr 호출? |
|---|---|
| 한글(.hwp/.hwpx) 파일 생성 | ✅ |
| 변형 문항/해설을 한글 수식 코드로 출력 | ✅ |
| `solve_prompt.txt`·`variant_solve_prompt.txt` 수정 | ✅ |
| `backend/data/hwp_math_mapping.json` 편집 | ✅ |
| `[ ]` 안의 코드 검토 | ✅ |
| 풀이 한국어 서술 검토 (용어·문체) | ❌ → `hsr` |
| 그래프 SVG 렌더 결함 진단 | ❌ → `docs/그래프_렌더_규칙.md` |

## 핵심 9 규칙 (가장 자주 위반)

| # | 규칙 | 예시 |
|---|---|---|
| 1 | 모든 숫자·알파벳은 `[ ]` | `함수 [f]의 개수는 [512]` |
| 2 | 변수 선언 시 각각 분리 | `두 실수 [a], [b]에 대하여` |
| 3 | 식 변형(등호 연속)은 분리, 방정식은 안 분리 | `[(1+i)^2][=1+2i+i^2][=2i]` vs `[ax^2+bx+c=0]` |
| 4 | ω = `omega` (★ `w` 금지) | `[omega^3 =1]` |
| 5 | 각도 = `DEG` (대문자, ★ `deg` 금지) | `[60 DEG]` |
| 6 | 사용자 정의 이항연산 별표 = `∗` (U+2217, ★ `*` 금지) | `[A∗B=2A-B^2]` |
| 7 | 중복조합·중복순열 = `smallprod` (★ `Pi` / `rmH` 금지) | `[{}_{4} smallprod _{4}]` |
| 8 | ± = `+-` (★ `pm` 금지) | `[x= {-b +- sqrt{b^2 -4ac}} over {2a}]` |
| 9 | 함수 표기 콜론·화살표: `[f`:`X ~->~ Y]` | 콜론 앞뒤 백틱, 화살표 앞뒤 틸더 |

## 자주 쓰는 패턴

```
복소수:     [z=a+b i]              ← 항상 b와 i 사이 띄어쓰기
허수단위:   [a+b i] (★ [a+bi] 금지)
켤레복소수: [bar{z`}]
극한:       [lim_{x`->`a}`{f left(x right) }]
정적분:     [int_{a}^{b} f left(x right) `dx]
시그마:     [sum_{k=1}^{n} {k^{2}}]
조합 맨앞:  [{}_{6} C _{2}]        ← 맨 앞 빈 중괄호 필수
도형 점:    [rmA], [rm{bar{AB}}], [rm{angle{ABC}}]
직각:       [rm{angle{ABC}} =it 90 DEG]
```

## 호출 후 행동

1. **SOT 본체 (`한글 수식입력기_260425.txt`)를 Read 로 열어** 작업 중인 코드 케이스에 해당하는 절을 확인.
2. 작업 직후 SOT 17장 체크리스트를 작업한 코드에 대조.
3. 새 결함 패턴 발견 시 SOT 본체 갱신 → 본 skill 의 빠른참조표도 갱신 (한 짝).
