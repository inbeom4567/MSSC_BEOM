# 고등수학 풀이 작성 규칙 (hsr)

이 skill은 **풀이를 어떻게 서술할까** (용어·논리 흐름·문체)를 다룬다.
수식 코드 표기는 `hmr` skill.

## 단일 정본 (SOT)

📄 **`docs/고등수학_풀이_규칙.md`**

본 skill 은 정본의 트리거·핵심 요약·자주 보는 빠른참조표일 뿐이다.
편집·확장은 항상 SOT 본체에서 한다 (역방향 금지).
**학년별·단원별 누적 구조** — 새 케이스가 발견되면 SOT 본체에 추가.

## 언제 호출해야 하는가

| 상황 | hsr 호출? |
|---|---|
| 수학 문제 풀이/해설 본문 한국어 서술 | ✅ |
| 변형 문항 풀이를 처음 작성 | ✅ |
| 해설 검수 (용어·문체) | ✅ |
| `solve_prompt.txt` 풀이 흐름·용어 부분 수정 | ✅ |
| 학년별·단원별 풀이 패턴 누적 | ✅ |
| 한글 수식 코드 `[ ]` 안 검토 | ❌ → `hmr` |
| 한글 .hwpx 파일 자체 생성 (코드 출력) | hmr 우선, 본 skill 보조 |

## 핵심 6 규칙 (가장 자주 위반)

### 1. 그림에서 답을 직접 읽지 말 것 ★
> ❌ "그래프에서 [a=2]임을 알 수 있다"
> ✅ 미지수 도입 → 조건 적용 → a 결정 → f(x) 결정

### 2. 용어 통일 ★

| 사용 금지 | 올바른 표현 |
|---|---|
| 위로 열린 / U자 모양 | 아래로 볼록 |
| 아래로 열린 / ∩자 모양 | 위로 볼록 |
| 실수부 / 허수부 | 실수부분 / 허수부분 |
| 지표 / 가수 | 정수부분 / 소수부분 |
| log_e | 자연로그 (`ln` 표기) |

### 3. 사잇소리 'ㅅ' 빠뜨리지 않기

최댓값 · 최솟값 · 극댓값 · 극솟값 · 함숫값 · 꼭짓점 · 기댓값 · 절댓값 · 근삿값 · 초깃값

### 4. 장식 기호 금지

❌ ★ ✓ → (강조용 화살표) — 풀이 본문에 사용 금지
✅ "따라서 ~이다", "그러므로 ~이다" 자연스러운 연결어

### 5. 비격식 표현 금지

| 비격식 | 권장 |
|---|---|
| ~로 두자 | ~라 하자 |
| ~을 박으면 | ~을 대입하면 |
| 풀어보면 | 풀면 |

### 6. 단계별 서술 표준 흐름

1. 조건 분석 → 식 세우기
2. 식 정리 → 미지수 결정
3. 결과 계산
4. (필요 시) 검산

## 자주 쓰는 풀이 패턴

| 그래프 | 미지수 | 결정 조건 |
|---|---|---|
| 이차함수 | [a left(a ne 0 right)] | x절편 + y절편 |
| 일차함수 | [m left(m ne 0 right)], [n] | 두 점 |
| 삼각함수 | 진폭·주기 | 최댓값·주기 |
| 지수함수 | [a left(a>0,~a ne 1 right)] | 한 점 |

## 호출 후 행동

1. **SOT 본체 (`docs/고등수학_풀이_규칙.md`)를 Read** — 단원·케이스에 해당하는 절 확인.
2. 작업 직후 SOT 5장 체크리스트로 자가 검수.
3. 새 단원·케이스 발견 시:
   - SOT 본체 3장(학년별 단원 범주)에 추가
   - 일반 원칙이면 1~2장으로 승격
   - 풀이 패턴이면 4장으로
   - 변경 이력 한 줄 추가
4. 풀이 코드 출력으로 넘어갈 때는 `hmr` skill 도 같이 호출.
