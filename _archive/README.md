# `_archive/` — 보관 자료실

이 폴더는 **현재 프로젝트에서 더 이상 사용하지 않지만, 후일 참조할 가능성이 있어
삭제하지 않고 보관**하는 자료를 모아 두는 곳입니다.

## 운영 원칙

1. **배포 자동 제외** — 본 폴더는 프로젝트 루트에 위치하며,
   `docker-compose.yml`은 `./backend` / `./frontend` 두 폴더만 빌드 컨텍스트로
   사용하므로, `_archive/` 는 어떤 Docker 이미지에도 자동으로 포함되지 않는다.
   (별도 `.dockerignore` 설정 불필요)

2. **삭제 금지 — 이동만** 허용. 보관물을 빼낼 때는 git 히스토리에 이동 사유를
   commit message로 남길 것.

3. **하위 폴더 명명** — `YYYY-MM-DD_주제` 형식. 같은 날짜에 여러 주제가 모이면
   주제별 하위 폴더 분리.

4. **각 보관 항목은 본 README에 보관 목적·이유·교체 후속물을 반드시 기록**.

5. **백엔드·프론트엔드가 실제 사용 중인 파일은 옮기지 않는다.** 예:
   - `backend/prompts/solve_prompt.txt` (ClaudeService가 매 요청마다 로드)
   - `backend/prompts/variant_solve_prompt.txt`
   - `backend/data/hwp_math_mapping.json`
   - `.claude/skills/hmr.md`
   이들은 **활성 자산**이며 archive 대상이 아니다.
