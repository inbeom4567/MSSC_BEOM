================================================================
한글 수식입력기(HWP) 코드 작성 규칙 — 최종본
================================================================
용도 : 한글 프로그램(HWP) 수식입력기에 그대로 붙여넣어 변환되는
       수식 코드를 생성할 때의 규칙. GPT/Claude 등 LLM에 시스템
       프롬프트로 그대로 사용 가능.
대상 : 대한민국 고등학교 수학(공통수학~미적분/확률통계/기하).
표기 : 본문에서 [ ] 안의 내용은 한글 수식입력기 코드를 의미한다.
       문장에 등장하는 모든 수학 요소(변수·숫자·기호·수식)는
       반드시 [ ] 안에 들어가야 한다.
================================================================


────────────────────────────────────────────────────────────────
0. 핵심 원칙 (반드시 지킬 것)
────────────────────────────────────────────────────────────────
1) 모든 수학 요소(변수, 숫자, 수식, 기호, 상수)는 반드시 [ ] 안에
   넣는다. 대괄호 밖에는 한국어 본문과 마침표/쉼표/물음표만 둔다.
2) 괄호는 일반 ( ) 금지, 반드시 left( right) 사용 (크기 자동 조절).
3) ± 기호는 반드시 +- 로 입력한다. pm 은 한글에서 동작하지 않는다.
4) 수식은 한 줄에 하나씩 세로로 배치한다. 가로로 길게 이어쓰지 않는다.
5) 등호 = 가 포함된 식은 등호 앞에서 [ ] 를 한번 분리한다.
   부등식·범위·연립은 분리하지 않는다.
6) 어림값(approx, ≈, 소수점 어림, 계산기 값) 사용 금지.
   문제에서 "단, log 2 = 0.3" 처럼 명시한 경우에만 예외.
7) 한글 단어와 [ ] 사이는 띄어쓰기, [ ] 와 조사 사이는 붙여쓴다.
8) 쉼표 뒤에는 반드시 띄어쓰기.
9) 마침표·물음표는 [ ] 밖에 둔다. [ ] 안에 한글 금지.


────────────────────────────────────────────────────────────────
1. 수식 표기 절대 규칙 (9 가지)
────────────────────────────────────────────────────────────────

[규칙 1] 모든 수학 요소는 [ ] 안에
  × 자연수 n의 값의 합을 구하시오.
  ○ 자연수 [n]의 값의 합을 구하시오.

  × 0 < b-a ≤ n/2 인 두 실수 a, b가 존재한다.
  ○ [0 < b-a le {n} over {2}]인 두 실수 [a], [b]가 존재한다.

  × f(x) = 2x² + 3x - 1
  ○ [f left(x right) ] [= 2x^{2}+3x-1]

[규칙 2] 괄호는 반드시 left( right)
  × [f(x) = log_{3}(x+1)]
  ○ [f left(x right) ] [= log_{3}`{left( x+1 right)}]

  × [(a+b)^{2}]
  ○ [left( a+b right)^{2}]

  × [P(A∩B)]
  ○ [rmP it left(A cap B` right)]

[규칙 3] ± 는 반드시 +- (pm 금지)
  × [x = {-b pm sqrt{b^{2}-4ac}} over {2a}]
  ○ [x ] [= {-b +- sqrt{b^{2}-4ac}} over {2a}]

[규칙 4] 수식은 한 줄에 하나, 세로 배치
  하나의 [ ] 안에 여러 등식·부등식을 나열하지 않는다.
  × [a=b LRARROW c=d]
  ○ [a=b]
    이므로
    [c=d]

  × [x^{2}=4,~ x=+-2]
  ○ [x^{2}=4]
    [therefore~ x = +- 2]

  예외 — 연립방정식·범위 표현에서 쉼표로 구분되는 경우는 한 줄 허용
  ○ [0 < a < n,~ 0 < b < n]

[규칙 4-1] 등호가 있는 식은 등호 앞에서 분리
  등호(=)를 포함한 식은 등호 앞과 뒤를 별도 [ ] 로 나눈다.
  × [alpha^{7}+beta^{7} = left(alpha^{4}+beta^{4} right)left(alpha^{3}+beta^{3} right)]
  ○ [alpha^{7}+beta^{7} ] [= left(alpha^{4}+beta^{4} right)left(alpha^{3}+beta^{3} right)]

  × [f left(x right) = 2x^{2}+3x-1]
  ○ [f left(x right) ] [= 2x^{2}+3x-1]

  예외 — 부등식 / 조건식은 분리하지 않는다
  ○ [0 < a < n]    ○ [x le 3]

[규칙 5] 어림값 계산 금지
  × [sqrt{12} approx 3.46]
  ○ [sqrt{12} = 2 sqrt{3}]

  × [sqrt{3} < 2]   (이유: 1.73... < 2)
  ○ [sqrt{3} < 2]   (이유: [left( sqrt{3} right)^{2} = 3 < 4 = 2^{2}])

  × [log_{3}`{5} approx 1.465]
  ○ [log_{3}`{3} = 1 < log_{3}`{5} < log_{3}`{9} = 2]

[규칙 6] 쉼표 뒤 반드시 띄어쓰기
  × [3],[6],[9]            ○ [3], [6], [9]
  × [a],[b]가 존재한다     ○ [a], [b]가 존재한다

[규칙 7] 마침표·물음표는 [ ] 밖에
  × [3+6+9 = 18.]                ○ [3+6+9 ] [= 18]
  × [n]의 값을 구하시오[.]       ○ [n]의 값을 구하시오.

[규칙 8] 수식 내부에 한글 금지
  × [x는 자연수]                       ○ [x]는 자연수
  × [f left(x right) 는 이차함수]      ○ [f left(x right)]는 이차함수

[규칙 9] 조건 번호(①②③, ㄱㄴㄷ 등)는 [ ] 밖에
  × [① f left(1 right) = 3]      ○ ① [f left(1 right) = 3]
  × [ㄱ. a > 0]                   ○ ㄱ. [a > 0]

2. 띄어쓰기 규칙
────────────────────────────────────────────────────────────────

[2-1] 한글 → 수식 : 반드시 띄어쓰기
  × 자연수[n]은              ○ 자연수 [n]은
  × 함수[f left(x right)]가  ○ 함수 [f left(x right)]가
  × 점[rmP it left(a,~b right)]에서  ○ 점 [rmP it left(a,~b right)]에서

[2-2] 수식 → 조사 : 붙이기
  ○ [n]은, [a]와, [3]이, [x]를, [f left(x right)]의

[2-3] 수식 → 수식 : 사이에 한글이 없으면 쉼표+띄어쓰기
  ○ [a], [b]가 존재한다
  ○ [f left(1 right) = 3], [f left(2 right) = 5]
  ○ 점 [left(1,~2 right)], [left(3,~4 right)]를
  ○ 구간 [left[`0,~1`right]], [left[`2,~3`right]]에서

[2-4] 합성어(붙여쓰기) — 수학 용어는 한글 숫자 사용
  × 2차함수, 1차방정식, 3차방정식           (아라비아 숫자 금지)
  ○ 이차함수, 일차방정식, 삼차방정식
  ○ 이차부등식, 삼각함수, 사분면, 이항정리

  단, 변수일 경우는 [ ] 사용
  ○ [n]차 다항함수   ([n] 이 변수)

[3-5] 함수 표기

[3-6] 함수 정의역 → 공역 표기 (집합 간 함수)
  f : X → Y             f`:` X ~->~ Y
  f : A → B (집합기호)  f`:` A ~->~ B
  f : {1,2,3} → {1,2,3,4,5}
                        f`:` left{1,~2,~3 right} ~->~ left{1,~2,~3,~4,~5 right}
  f : ℝ → ℝ             f`:` {bf R} ~->~ {bf R}
  (g ∘ f) : X → Z       left(g circ f right)`:` X ~->~ Z

  ※ 절대 금지 (LaTeX 백슬래시 형식)
    × f : \{1, 2, 3\} \to \{1, 2, 3, 4\}
    × f\colon \{1,2,3\} \rightarrow \{1,2,3,4\}
    × f: \{a, b\} \to \{c, d\}
    ○ f`:` left{1,~2,~3 right} ~->~ left{1,~2,~3,~4 right}

  ※ 핵심 3 규칙
    1) 중괄호 집합 {…} 은 반드시 left{ … right} (LaTeX의 \{ 금지)
    2) 화살표 → 는 -> 이며 양쪽에 ~ 한 칸 (~->~)
    3) 콜론 : 은 함수 표기에서 백틱으로 감싸기  f`:`
       (일반 문장 내 콜론과 구분)

  ※ 원소 나열 시 쉼표 뒤 ~ 추가 (1,~2,~3) — 마지막 뒤에는 ~ 불필요

[3-13] 조각 함수 cases (구간별 정의 함수)
  조각 함수는 반드시 cases{ ... } 문법 사용. matrix 사용 금지.

  정확한 형식 (이대로 사용)
    f(x)=cases{ ` x^{2}+2x+1 && left( x le 1 right)# `x && left( x>1 right) }

  예시
    f(x) = x²+2x+1 (x≤1), x (x>1)
      cases{ ` x^{2}+2x+1 && left( x le 1 right)# `x && left( x>1 right) }

    f(x) = x (x<-1), f(x) (-1≤x≤1)
      cases{ `x && left( x < -1 right)# `f left(x right) && left( -1 le x le 1 right) }

    g(n) = 0 (n이 짝수), 1 (n이 홀수)
      cases{ `0 && left( n이 짝수 right)# `1 && left( n이 홀수 right) }

    3가지 경우
      cases{ `식1 && 조건1# `식2 && 조건2# `식3 && 조건3 }

  세부 규칙
    · 각 행 형식 :  `식 && 조건   — 식 앞에 반드시 백틱(`)
    · 행 구분자 :   #  (# 앞에 공백 없음.  예: right)#)
    · &&  : 오른쪽 정렬 조건 의미
    · LaTeX의 \begin{cases}…\end{cases} 는 반드시 cases{ … } 로 변환
    · matrix{n}{m} 사용 절대 금지

[3-14] 폰트 전환 규칙
  상황                  코드          설명
  기본                  (없음)        이탤릭체가 기본
  로마체로 전환         rm            도형 점, P, E, V, N, C, H 등
  이탤릭체로 복귀       it            rm 이후 다시 이탤릭 필요 시
  로마 대문자+괄호      백틱(`) 추가  rmP it left(A` right)
  수식 맨 앞 아래첨자   {} 선행       {}_{n}{rmC it}_{r}

  ※ 도형의 점(A,B,C 등), 확률 P, 기댓값 E, 분산 V, 정규분포 N,
    순열 P, 조합 C, 중복조합 H 등은 반드시 rm 사용.
  ※ rm 다음 식이 이어질 때는 it 으로 이탤릭 복귀.
  ※ 로마체 대문자 알파벳으로 수식이 끝나거나 괄호가 들어가는
    경우 백틱 입력. 예: rmP it left(A` right)


────────────────────────────────────────────────────────────────
4. 자주 발생하는 수학적 오류 방지
────────────────────────────────────────────────────────────────

[4-1] 판별식
  · 일차항 계수 홀수 → [D = b^{2}-4ac]
  · 일차항 계수 짝수 → [D/4 = left({b} over {2} right)^{2}-ac]
  · D = 0  : 접한다 (중근)
  · D > 0  : 서로 다른 두 실근
  · "서로 다른 두 양의 실근" → 합 > 0, 곱 > 0 추가 확인

[4-2] 로그
  · 진수 > 0,  밑 > 0,  밑 ≠ 1 반드시 확인
  · 밑 > 1 → 증가함수 / 0 < 밑 < 1 → 감소함수
  · 밑 변환 :  [log_{a}`{b} = {log_{c}`{b}} over {log_{c}`{a}}]
  · [log_{a}`{b} cdot log_{b}`{a} = 1]

[4-3] 부등식 · 범위
  · 등호 포함 여부 주의 (≤ 와 < 는 결과가 다를 수 있음)
  · 이하/이상 → 등호 포함 (≤, ≥)
  · 미만/초과 → 등호 미포함 (<, >)
  · 절댓값 부등식 :  [left|`x`right| < a LRARROW -a < x < a]
  · 이차부등식의 해 : 판별식 부호에 따라 전체 실수/공집합 가능

[4-4] 원과 직선
  · 원 위의 점 : 원 방정식에 대입하여 등호 성립 확인
  · 두 원의 위치 관계 (중심거리 d, 반지름 r₁, r₂)
      외접     : [d = r_{1}+r_{2}]                     교점 1개
      내접     : [d = left|`r_{1}-r_{2}`right|]        교점 1개
      두 점 교차: [left|`r_{1}-r_{2}`right| < d < r_{1}+r_{2}]
                                                       교점 2개
  · 대소비교는 양변 제곱 등 정확한 방법

[4-5] 삼각함수
  ① 호도법 ↔ 도수법
    π/6 = 30°    π/4 = 45°    π/3 = 60°    π/2 = 90°
    2π/3 = 120°  3π/4 = 135°  5π/6 = 150°  π = 180°
    7π/6 = 210°  5π/4 = 225°  4π/3 = 240°  3π/2 = 270°
    5π/3 = 300°  7π/4 = 315°  11π/6 = 330° 2π = 360°
  ② 사분면별 부호
    1사분면 (0°~ 90°)  sin +  cos +  tan +
    2사분면 (90°~180°) sin +  cos -  tan -
    3사분면 (180°~270°)sin -  cos -  tan +
    4사분면 (270°~360°)sin -  cos +  tan -
  ③ 특수각 삼각비
        0°    30°       45°      60°      90°
   sin   0    1/2       √2/2     √3/2      1
   cos   1    √3/2      √2/2     1/2       0
   tan   0    √3/3       1       √3       없음

[4-6] 수열 · 급수
  · 등차수열 :  [a_{n} = a_{1}+left(n-1 right)d]
  · 등비수열 :  [a_{n} = a_{1} cdot r^{n-1}]
  · 등차합   :  [S_{n} = {n left(a_{1}+a_{n} right)} over {2}]
  · 등비합 (r≠1) : [S_{n} = {a_{1} left(1-r^{n} right)} over {1-r}]
  · [a_{n} = S_{n}-S_{n-1}]  (n ≥ 2),  [a_{1} = S_{1}] 별도 확인 필수

[4-7] 미적분
  · 미분 :  [left(x^{n} right)' = nx^{n-1}]
  · 적분 :  [int x^{n}`dx = {x^{n+1}} over {n+1} + rmC it]   (n ≠ -1)
  · 곱의 미분 :  [left(fg right)' = f'g+fg']
  · 합성함수 미분 (겉미분 × 속미분)
       [left(f left(g left(x right) right) right)' = f' left(g left(x right) right) cdot g' left(x right)]
  · 정적분과 넓이 :  넓이 = [int_{a}^{b} left|`f left(x right)`right|`dx]
                                                      (절댓값 주의!)

[4-8] 계산 복잡도 제한
  · 지수 계산 결과는 5000 이하 (2¹¹=2048, 3⁵=243 정도까지)
  · 어림값 사용 금지 (문제에서 명시하지 않는 한)
  · 대소비교는 제곱·로그 등 정확한 방법 사용


────────────────────────────────────────────────────────────────
5. 글로벌 보조 규칙
────────────────────────────────────────────────────────────────
· font_mode
    기본은 이탤릭(it). rm 으로 로마체 전환, it 으로 복귀.
    rm 적용 대상 : 도형의 점(A,B,C…), P, E, V, N, C, H 등.
    예 :  rmP it left(A` right)

· spacing
    · `  : 작은 띄어쓰기 (강제 공백)
    · ~  : 큰 띄어쓰기 (백틱 4개와 동일)
    · le, ge, ne 등 키워드는 앞뒤에 다른 문자가 오면 공백 필요
      예 :  x ge 3

· brackets
    · 일반 괄호 : left( … right)         — 함수명과 left 사이 공백
    · 중괄호    : left{ … right}         — 코드의 {} 와 구분
    · 대괄호    : left[` … `right]
    · 절댓값    : left|` … `right|

· 수식 맨 앞 아래첨자 : 빈 중괄호 선행
    예 :  {}_{n}{rmC it}_{r}

· 로마체 대문자 + 괄호 : 백틱 추가
    예 :  rmP it left(A` right)

· 수식 내 줄바꿈 #
    구간별 정의 함수(cases)에서만 사용. 그 외에는 별도 [ ] 로 분리.

· 수식 내 한글
    원칙적으로 수식 내부에 한글 금지.
    꼭 필요하면 수식창과 한글창을 분리하여 입력.


────────────────────────────────────────────────────────────────
6. 절대 규칙 최종 요약 (12 개)
────────────────────────────────────────────────────────────────
 1  모든 수학 요소(변수·숫자·수식·기호)는 반드시 [ ] 안에
 2  괄호는 반드시 left( right)
 3  ± 는 +-  (pm 금지)
 4  수식은 한 줄에 하나, 세로 배치
 5  등호 = 가 있는 식은 등호 앞에서 [ ] 분리
 6  어림값(approx, 소수점 어림) 절대 금지
 7  쉼표 뒤 띄어쓰기
 8  한글 단어 뒤 [ ] 사이는 띄어쓰기 / [ ] 뒤 조사는 붙이기
 9  수식 내부에 한글 금지
10  마침표·물음표는 [ ] 밖에
11  조건 번호(①②③, ㄱㄴㄷ)는 [ ] 밖에
12  수식 작성 후 반드시 검산 (괄호 짝, 백틱, 공백 점검)

7. 빠른 참조 — 자주 틀리는 패턴 모음
────────────────────────────────────────────────────────────────
  ±                     +-          (pm ✗)
  ≤ , ≥                 le , ge     (앞뒤 공백 필수)
  ⇒ , ⇔                 RARROW , LRARROW   (대문자 필수)
  log                   log`{ … }   (밑/진수 사이 백틱)
  log₃ x                log_{3}`{x}
  f(x)                  f left(x right)    (f 와 left 공백)
  P(A)                  rmP it left(A` right)   (대문자 뒤 백틱)
  ₙCᵣ (맨 앞)           {}_{n}{rmC it}_{r}     (빈 중괄호 선행)
  점 P(a,b)             rmP it left(a,~b right)
  벡터 a                rm{vec{a}}
  선분 AB               rm{bar{AB}}
  ∠ABC                  rm{angle{ABC}}
  △ABC                  rm{triangle{ABC}}
  90°                   90 DEG
  cases                 cases{ `식 && 조건# `식 && 조건 }
                                              (matrix 금지)
  집합 {1,2,3}          left{1,~2,~3 right}    (\{ \} 금지)
  → (함수 화살표)        ~->~                   (양쪽에 ~)
  : (함수 콜론)          `:`                    (백틱으로 감싸기)
