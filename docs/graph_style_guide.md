# 수학 교과서 그래프 스타일 가이드

기준 파일 `무제-1.svg`(좌표평면 단독) + 4권 책 그림 SVG (`_1_고등수학`, `_2_대수`, `_3_미적분1`, `_4_미적분2`)에서 추출한 **표준 좌표평면 규격**입니다.

## 1. 좌표축 라인 규격

| 속성 | 값 | 비고 |
|---|---|---|
| `stroke` | `#231815` | 다크 그레이 (순흑 #000 아님) |
| `stroke-width` | `0.75px` | 좌표축 표준. 보조 점선은 0.5px |
| `stroke-miterlimit` | `10` | 일러스트레이터 기본값 |

> **주의**: 일러스트레이터가 출력하는 `#231815`는 CMYK 기반의 K100 환산색입니다. 화면상 검정과 거의 구분 안 되지만, 표준을 지키려면 이 값 사용을 권장합니다.

## 2. 화살촉 (Arrow Head) 규격

4개 책 SVG 모두 **가장 빈번하게 등장하는 사이즈가 동일**합니다.

| 항목 | 값 |
|---|---|
| **너비 (단축)** | `2.78` |
| **길이 (장축)** | `3.29` |
| **장축 / 단축 비율** | `1.18` |
| **오목 깊이 (안쪽 V)** | `0.59` |
| **라인 끝 ~ 화살촉 꼭짓점 거리** | `2.24` |

### 2-1. 화살촉 polygon 좌표 공식

화살촉은 **4점 polygon (마지막에 첫 점 반복하여 5쌍으로 표기)** 입니다.

#### 위 방향 화살촉 (y축 끝)
꼭짓점 좌표 `(tx, ty)` 기준:
```
(tx+1.39, ty+3.29)   ← 오른쪽 밑
(tx,      ty+2.70)   ← 안쪽 오목 (V자 들어간 부분)
(tx-1.39, ty+3.29)   ← 왼쪽 밑
(tx,      ty)        ← 꼭짓점 (가장 위)
(tx+1.39, ty+3.29)   ← 시작점 반복
```

#### 오른쪽 방향 화살촉 (x축 끝)
꼭짓점 좌표 `(tx, ty)` 기준:
```
(tx-3.29, ty+1.39)   ← 왼쪽 아래
(tx-2.70, ty)        ← 안쪽 오목
(tx-3.29, ty-1.39)   ← 왼쪽 위
(tx,      ty)        ← 꼭짓점 (가장 오른쪽)
(tx-3.29, ty+1.39)   ← 시작점 반복
```

> 아래/왼쪽 방향은 위 좌표를 부호 반전하면 됩니다.

### 2-2. 라인과 화살촉의 관계

라인은 화살촉 꼭짓점 끝까지 그리지 **않습니다**. 라인은 화살촉 밑변 안쪽 약간 못 미친 곳에서 끝나고, 그 위에 화살촉을 얹어둡니다.

| 항목 | 값 |
|---|---|
| 라인 끝점 → 화살촉 밑변 (오목 안쪽) | `0.46` |
| 라인 끝점 → 화살촉 꼭짓점 | `2.24` |

**구체적 예시 (y축, 원점 (94.87, 86.05)):**
- 라인 끝: `(94.87, 23.31)`
- 화살촉 꼭짓점: `(94.87, 21.07)`
- 차이: `2.24` (라인은 화살촉 안쪽 0.46 만큼 들어가고, 거기서 2.24 더 가서 꼭짓점)

## 3. 축 라벨 (x, y, O)

| 항목 | 값 |
|---|---|
| 폰트 | `HyhwpEQ` (한글 수식 전용 폰트) |
| 크기 | `8.99px` |
| 색상 | `#231815` (`fill`) |

### 3-1. 핵심 발견: HyhwpEQ는 PUA 매핑 폰트

**HyhwpEQ는 ASCII 영역(0x21~0x7F)에 글리프가 없습니다.** 한글 수식편집기가 사용하는 모든 글자는 PUA(사용자 영역, `0xE000~0xE10F`)에 매핑되어 있습니다. 이는 같은 글자도 **로마체와 이탤릭이 별도의 코드 포인트**를 갖는다는 뜻입니다.

따라서 SVG에 단순히 `<tspan>x</tspan>`이라고 쓰면 HyhwpEQ가 렌더링하지 못하고 시스템 폴백 폰트(예: HCR Batang)로 표시됩니다. **이탤릭으로 보이게 하려면 PUA 코드 포인트를 직접 사용**해야 합니다.

### 3-2. HyhwpEQ PUA 매핑표

| 영역 | 코드 범위 | 설명 |
|---|---|---|
| **로마체 대문자** A~Z | `0xE000` ~ `0xE019` | 점 이름 (O, A, B, P 등) |
| **로마체 소문자** a~z | `0xE01A` ~ `0xE033` | 함수명, 단위 등 |
| **로마체 숫자** 0~9 | `0xE034` ~ `0xE03D` | |
| **그리스 대문자** Α~Ψ | `0xE084` ~ `0xE09A` | (대부분 영문과 모양 같음) |
| **그리스 소문자 (이탤릭)** α~ω | `0xE09B` ~ `0xE0B2` | α=E09B, β=E09C, ..., θ=E0A2, π=E0AA, σ=E0AC, ω=E0B2 |
| **이탤릭 대문자** A~Z | `0xE0CC` ~ `0xE0E4` | 블랙보드/이중라인 형태 (ℝ, ℂ 등) |
| **이탤릭 소문자** a~z | `0xE0E5` ~ `0xE0FE` | **변수용 (x=E0FC, y=E0FD)** |

### 3-3. 라벨 종류별 처리

| 라벨 | 폰트 코드 | 예시 |
|---|---|---|
| 변수 (`x`, `y`, `t`, `n`) | 이탤릭 PUA (E0E5~) | `x` → `chr(0xE0FC)` |
| 그리스 변수 (`θ`, `π`, `α`) | 이탤릭 PUA (E09B~) | `θ` → `chr(0xE0A2)` |
| 점 이름 (`O`, `A`, `B`, `P`) | 로마체 PUA (E000~) | `O` → `chr(0xE00E)` |
| 숫자 (`0`, `1`, `2`) | 로마체 PUA (E034~) | `0` → `chr(0xE034)` |

### 3-4. 변환 함수 (Python)

```python
ITALIC_LOWER_BASE = 0xE0E5  # a → E0E5, x → E0FC, y → E0FD, z → E0FE
ROMAN_UPPER_BASE  = 0xE000  # A → E000, O → E00E, P → E00F

def hwp_var(label):
    """변수 라벨을 HyhwpEQ 이탤릭 PUA 코드로 변환"""
    if len(label) == 1 and 'a' <= label <= 'z':
        return chr(ITALIC_LOWER_BASE + ord(label) - ord('a'))
    return label

def hwp_point(label):
    """점 이름을 HyhwpEQ 로마체 PUA 코드로 변환"""
    if len(label) == 1 and 'A' <= label <= 'Z':
        return chr(ROMAN_UPPER_BASE + ord(label) - ord('A'))
    return label
```

### 3-5. CSS

```css
.axis-text { font-family: HyhwpEQ, "HCR Batang", serif; font-size: 8.99px; }
```

> 폰트 폴백으로 `HCR Batang`, `serif`를 두면 HyhwpEQ가 깔리지 않은 환경에서도 비슷한 모양으로 표시됩니다. **`font-style: italic`은 필요 없습니다** (PUA 코드 자체가 이미 이탤릭 글리프를 가리키므로).

### 3-6. O 라벨 위치

| 항목 | 값 |
|---|---|
| 원점 대비 x 오프셋 | `-8.81` |
| 원점 대비 y 오프셋 | `+10.90` (아래쪽) |

**위치 의미**: O는 원점의 **왼쪽 아래** (3사분면 쪽 약간 떨어진 자리)에 놓습니다.

폰트 크기 대비 비율:
- x 오프셋: 폰트 크기의 **약 -1.0배**
- y 오프셋: 폰트 크기의 **약 +1.2배**

> SVG에서 `<text transform="translate(...)">`의 좌표는 텍스트 베이스라인 좌표입니다.

## 5. 점선 (Dashed Line) 규격

보조선·점선은 좌표축보다 가늘게, 짧은 대시 패턴을 씁니다.

| 용도 | stroke-dasharray | stroke-width |
|---|---|---|
| 짧은 점선 (수직선 보조) | `1 1` | `0.5px` |
| 표준 점선 | `1.5 1.5` (또는 6쌍 반복) | `0.5px` |
| 살짝 긴 점선 | `2 2` 또는 `3 2` | `0.5px` |

일러스트레이터 출력 시 패턴이 6쌍으로 반복되는 경우 (`1.5 1.5 1.5 1.5 1.5 1.5`)가 흔하지만, 의미상 `1.5 1.5`와 동일합니다.

## 10. graph_builder 사용법

`graph_builder.py`의 `Graph` 클래스로 좌표평면 위에 함수 곡선·점·점선을 추가할 수 있습니다.

### 10-1. 기본 사용

```python
from graph_builder import Graph

g = Graph(width=300, height=200, scale=20)   # scale: 1단위당 SVG 픽셀
g.plot(lambda x: x**2 - 2*x - 3, x_range=(-2, 4))
g.point(-1, 0, label='A', label_pos='below_left')
g.point( 3, 0, label='B', label_pos='below_right')
g.point( 1, -4, label='P', label_pos='below')

with open('output.svg', 'w', encoding='utf-8') as f:
    f.write(g.render())
```

### 10-2. Graph 생성자 매개변수

| 매개변수 | 기본값 | 설명 |
|---|---|---|
| `width`, `height` | `300, 220` | SVG 전체 크기 |
| `ox`, `oy` | 자동 | 원점 위치 (None이면 width\*0.48, height\*0.52) |
| `scale` | `20` | 좌표평면 1단위 = 몇 px |
| `x_label`, `y_label` | `'x', 'y'` | 축 라벨 (이탤릭 자동 변환) |
| `show_o` | `True` | 원점 O 표시 여부 |

### 10-3. 메서드

| 메서드 | 설명 |
|---|---|
| `g.plot(func, x_range, samples=120, color, stroke_width, dashed)` | 함수 곡선 |
| `g.point(x, y, label, label_pos, color, radius)` | 점 + 라벨 |
| `g.dashed_v(x, y_from, y_to)` | 수직 점선 (좌표 강조용) |
| `g.dashed_h(y, x_from, x_to)` | 수평 점선 |
| `g.line(x1, y1, x2, y2, dashed=False)` | 일반 직선 |
| `g.x_tick(x, label)` | x축 위 눈금 + 라벨 |
| `g.y_tick(y, label)` | y축 위 눈금 + 라벨 |
| `g.render()` | 완성된 SVG 문자열 반환 |

### 10-4. 라벨 위치 (`label_pos`)

점 라벨은 8방향으로 배치 가능:
- `'above'`, `'below'`, `'left'`, `'right'`
- `'above_left'`, `'above_right'`, `'below_left'`, `'below_right'`

### 10-5. 자동 처리되는 것들

- **클리핑**: 곡선이 그래프 영역을 벗어나면 SVG `<clipPath>`로 자동 잘라냄
- **PUA 변환**: 라벨에 'x', 'y', 'theta', 1, 2, '-' 등을 입력하면 자동으로 HyhwpEQ 코드로 변환
- **그리스 단어**: `'theta'`, `'pi'`, `'alpha'` 등의 단어 입력 시 그리스 글리프로 변환
- **정의역 끊김**: `1/x` 같이 정의역이 끊긴 함수도 안전하게 처리

### 10-6. 예시

| `demo_parabola.svg` | 포물선 + 두 교점(A, B) + 꼭짓점(P) |

## 11. HyhwpEQ PUA 전체 매핑 (요약)

`hwp_text(s)` 함수가 자동으로 처리하는 매핑:

| 종류 | 입력 | PUA 코드 |
|---|---|---|
| 이탤릭 소문자 | `'a'` ~ `'z'` | `0xE0E5` ~ `0xE0FE` |
| 로마체 대문자 (점 이름) | `'A'` ~ `'Z'` | `0xE000` ~ `0xE019` |
| 로마체 숫자 | `'0'` ~ `'9'` | `0xE03D`, `0xE034`~`0xE03C` |
| 마이너스 | `'-'` | `0xE046` |
| 등호 | `'='` | `0xE047` |
| 플러스 | `'+'` | `0xE048` |
| 괄호 | `'('`, `')'` | `0xE044`, `0xE045` |
| 대괄호 | `'['`, `']'` | `0xE049`, `0xE04A` |
| 중괄호 | `'{'`, `'}'` | `0xE04B`, `0xE04C` |
| 비교 연산자 | `'<'`, `'>'` | `0xE055`, `0xE056` |
| 그리스 (단어) | `'theta'`, `'pi'`, ... | `hwp_var()` 사용 |
