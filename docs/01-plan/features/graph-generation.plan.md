# Plan: 그래프 자동 생성 + 손필기/그림 인식 (graph-generation)

## 1. 개요

수학 문제에서 그래프/그림이 포함된 경우:
1. **원본 분석**: 원본 문제의 그래프/손필기/그림을 Gemini가 분석하여 수학적 의미 파악
2. **유사문항 그래프 생성**: 변형된 문제에 맞는 새로운 그래프를 Gemini 이미지 생성으로 자동 제작
3. **풀이 그래프 포함**: 풀이 과정에서 필요한 그래프도 자동 생성하여 포함

**핵심 기술**: Google Gemini API (나노바나나 = Gemini 이미지 생성 모델)

## 2. 요구사항

### 필수 (Phase 1)
- 원본 문제 이미지에서 그래프/그림을 Gemini가 분석 → 함수식, 도형, 좌표 등 추출
- 유사문항에 맞는 그래프를 Gemini 이미지 생성으로 자동 제작
- 생성된 그래프 이미지를 풀이 결과에 포함하여 표시
- 이차함수, 삼차함수, 삼각함수, 로그함수, 지수함수 등 고등학교 수준 그래프 지원

### 필수 (Phase 2)
- 손필기(손글씨) 이미지 인식 → 수식/텍스트 추출
- 원본 해설의 그래프 스타일을 참조하여 유사한 스타일로 생성

### 선택 (향후)
- 그래프 스타일 옵션 (격자, 색상, 축 범위 사용자 지정)
- 여러 함수를 하나의 그래프에 겹쳐 그리기
- 교점, 극값, 점근선 등 주요 포인트 자동 표시
- 도형 문제 (원, 삼각형, 사각형 등) 그림 생성

## 3. 기술 스택

| 구성요소 | 기술 | 비고 |
|---------|------|------|
| 이미지 분석 | Gemini API (generateContent + 이미지 입력) | 원본 그래프/손필기 분석 |
| 이미지 생성 | Gemini API (나노바나나 Pro/Flash) | 그래프/그림 생성 |
| 고품질 모델 | `gemini-3-pro-image-preview` | 정밀한 수학 그래프용 |
| 빠른 모델 | `gemini-3.1-flash-image-preview` | 간단한 그래프용 |
| 인증 | Google API Key (x-goog-api-key 헤더) | 발급 완료 |
| 이미지 형식 | PNG → base64 인코딩 | API 응답에서 직접 제공 |

## 4. Gemini API 사용 방법

### 4.1 엔드포인트
```
POST https://generativelanguage.googleapis.com/v1beta/models/{model}:generateContent
Header: x-goog-api-key: {GEMINI_API_KEY}
```

### 4.2 원본 이미지 분석 (그래프/손필기 인식)
```json
{
  "contents": [{
    "parts": [
      {"inline_data": {"mime_type": "image/png", "data": "<base64_원본이미지>"}},
      {"text": "이 수학 문제의 그래프를 분석해주세요. 함수식, 좌표, 교점, 축 범위를 정확히 추출해주세요."}
    ]
  }]
}
```

### 4.3 그래프 이미지 생성
```json
{
  "contents": [{
    "parts": [
      {"text": "수학 그래프를 그려주세요: y = x² - 4x + 3, x범위 -1~5, 꼭짓점 (2,-1) 표시, x절편 (1,0)(3,0) 표시, 깔끔한 교과서 스타일, 격자선 포함, 한글 라벨"}
    ]
  }],
  "generationConfig": {
    "responseModalities": ["TEXT", "IMAGE"]
  }
}
```

### 4.4 원본 참조 + 스타일 맞춤 생성
```json
{
  "contents": [{
    "parts": [
      {"inline_data": {"mime_type": "image/png", "data": "<base64_원본그래프>"}},
      {"text": "이 그래프와 같은 스타일로 y = 2x² - 8x + 6 의 그래프를 그려주세요. 동일한 격자 스타일, 색상, 라벨 방식을 유지하세요."}
    ]
  }],
  "generationConfig": {
    "responseModalities": ["TEXT", "IMAGE"]
  }
}
```

### 4.5 응답 처리
```json
{
  "candidates": [{
    "content": {
      "parts": [
        {"text": "그래프 설명..."},
        {"inline_data": {"mime_type": "image/png", "data": "<base64_생성이미지>"}}
      ]
    }
  }]
}
```

## 5. 구현 전략

### 5.1 전체 흐름

```
[원본 문제 이미지 업로드]
  → Claude가 문제 텍스트 분석 + 유사문항 생성
  → Claude 응답에 -그래프- 태그 포함 시:
    → Gemini API로 그래프 이미지 생성
    → 태그를 base64 이미지로 교체
  → 프론트엔드에 텍스트 + 이미지 혼합 결과 전달
```

### 5.2 Claude 출력 형식 (기존 유지)

Claude가 그래프가 필요한 곳에 특수 태그 삽입:
```
-그래프-
함수: y = x^2 - 4x + 3
x범위: -1, 5
포인트: (1, 0), (3, 0), (2, -1)
라벨: y = x² - 4x + 3
스타일: 교과서
-그래프끝-
```

### 5.3 손필기 인식 흐름

```
[손필기 이미지 업로드]
  → Gemini API로 이미지 분석 요청
  → 수식/텍스트 추출 결과 반환
  → Claude에게 추출된 내용 전달하여 문제 분석
```

### 5.4 원본 그래프 참조 흐름

```
[원본 문제에 그래프 포함된 경우]
  → Gemini API로 원본 그래프 분석 (함수식, 스타일 파악)
  → 유사문항 생성 시 분석된 정보를 Claude에게 전달
  → 변형된 함수에 대해 Gemini로 유사 스타일 그래프 생성
```

## 7. 구현 순서

### Phase 1: 기본 그래프 생성 (우선)
1. `gemini_service.py` 생성 — Gemini API 연동 (이미지 생성)
2. 프롬프트 수정 — Claude에게 -그래프- 태그 출력 규칙 추가
3. `claude_service.py` 수정 — 응답 후처리에서 그래프 태그 감지 → Gemini 호출
4. 프론트엔드 수정 — 그래프 이미지 렌더링
5. 테스트 — 이차함수, 삼각함수 등 다양한 그래프 검증

### Phase 2: 원본 분석 + 손필기 인식
6. `gemini_service.py` 확장 — 이미지 분석 기능 추가
7. 원본 그래프 분석 → 유사문항 그래프에 스타일 반영
8. 손필기 인식 API 엔드포인트 추가
9. 프론트엔드에 손필기 업로드 UI 추가

### Phase 3: 고도화
10. 원본 그래프 참조 스타일 맞춤 생성
11. 도형 문제 그림 생성
12. 그래프 품질 개선 (교과서 스타일 프롬프트 최적화)

## 9. 리스크 및 대응

| 리스크 | 대응 |
|--------|------|
| Gemini 그래프 정확도 부족 (눈금, 좌표 오차) | 프롬프트에 정확한 좌표/범위 명시 + 교과서 스타일 강조 |
| API 응답 시간 (이미지 생성 2-5초) | 스트리밍 UI로 "그래프 생성 중..." 표시 |
| Gemini가 한글 라벨을 잘못 렌더링 | 라벨은 영문/수식으로, 한글은 최소화 |
| API 비용 | Flash 모델 우선 사용, Pro는 고품질 필요 시만 |
| 손필기 인식 정확도 | Gemini Pro 모델 사용 + 인식 결과를 사용자에게 확인 요청 |
| base64 이미지 크기가 큼 | 적절한 해상도(1K) 사용, 프론트엔드에서 lazy loading |
