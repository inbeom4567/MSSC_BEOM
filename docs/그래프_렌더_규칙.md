## 0. 큰 그림

```
Claude(AI) ──→ "-그래프-...-그래프끝-" 명세 텍스트 (AI는 그림 안 그림)
              ↓
graph_service._parse → spec dict
              ↓
graph_builder.Graph (코드) → SVG 문자열
              ↓
resvg-py (코드) → PNG (viewBox 크기와 1:1)
              ↓
hwpx_service (코드) → BinData/graphN.png + content.hpf manifest + section0.xml <hp:pic>
```

## 규칙 2. 라벨 표기 ★ 절대 규정 ★

**가장 자주 위반된 규칙**. 라벨이 좌표축·곡선·점선과 겹쳐 가독성 깨지는 사고가 5번 이상.

### 2-1. 흰 후광 — 두 `<text>` 요소 분리 (paint-order 의존 금지!)

```xml
<text fill="white" stroke="white" stroke-width="12.6" stroke-linejoin="round">1</text>
<text fill="#231815">1</text>
```

| 속성 | 값 |
|---|---|
| 후광 두께 | `LABEL_FONT_SIZE × 0.35` (= 폰트 35.96 시 12.6px) |
| 위치 | 첫 text(흰 두꺼운 글자) → 두 번째 text(검정 글자)가 그 위에 |

**❌ 절대 금지**: `paint-order="stroke fill"` 단일 text — **resvg-py 가 무시**해 PNG 변환 시 후광 사라짐 (사고 발생 사례 있음, 2026-04-25 v4)

**✅ 정상**: 두 `<text>` 분리. 어느 PNG 변환기에서도 후광 보장.

**코드 위치**: `graph_builder.Graph._halo_text(x, y, anchor, text, halo_w)` 헬퍼

### 2-2. axis-라벨 거리

| 위치 | 거리 |
|---|---|
| x_tick (below) | `ly = py + LABEL_FONT_SIZE × 1.6` |
| x_tick (above) | `ly = py - LABEL_FONT_SIZE × 0.9` |
| y_tick 음수 | `ly = py + LABEL_FONT_SIZE × 0.45` |
| y_tick 양수 | `ly = py - LABEL_FONT_SIZE × 0.45 + 0.7×size` |
| y_tick 0 | `ly = py + LABEL_FONT_SIZE × 0.35` |
| y_tick 좌우 gap | `LABEL_FONT_SIZE × 0.35` |

**절대 규정**: 글자 위쪽이 axis 선에서 폰트의 0.9배 이상 떨어져야 한다 (글자 자체가 axis 선과 닿지 않음).

### 2-3. 빗겨가기 (음·양 방향에 따라)

x_tick:
- x < 0: 라벨 왼쪽 정렬(`anchor="end"`), `lx = px - LABEL_FONT_SIZE × 0.30`
- x > 0: 라벨 오른쪽 정렬(`anchor="start"`), `lx = px + LABEL_FONT_SIZE × 0.30`
- x = 0: 가운데 정렬

### 2-4. LaTeX 풍 라벨 변환 (`_smart_label_text`)

지원해야 하는 형식:
- `^x`, `^{...}` → `<tspan baseline-shift="super" font-size="70%">`
- `_x`, `_{...}` → `<tspan baseline-shift="sub" font-size="70%">`
- `sin/cos/tan/log/ln/exp(...)` → 로마체 PUA + thin space + 인자(괄호 자동 벗김)
- `pi`, `theta`, `\pi` → 이탤릭 그리스 PUA
- `2pi`, `3theta` 합성 → 숫자(로마) + 그리스
- 영문 소문자 → 이탤릭 PUA / 대문자 → 로마체 PUA (점 이름)

**❌ 위반 사례**:
- `^x` 가 `^` 캐럿 그대로 출력 (v4 이전)
- `sin(x)` 가 글자별 이탤릭 (v4 이전)
- `2pi` 가 `2pi` 글자 그대로 (v4 이전)

**✅ 정상**: 모두 `_smart_label_text` 통과 후 한글 수식입력기 표준 PUA 시퀀스로 변환

**코드 위치**: `graph_builder._smart_label_text(text)`

## 규칙 3. 곡선 스타일

| 속성 | 값 | 코드 |
|---|---|---|
| stroke 두께 | `1.0 × HWPX_SCALE` (= 4.0px) | `CURVE_STROKE_WIDTH` |
| 점선 패턴 | `1.5×SC 1.5×SC` | `DASHED_ARRAY` |
| 점 마커 반지름 | `1.4 × HWPX_SCALE` | `POINT_RADIUS` |

**y_clip 규정**: `plot()` 호출 시 `y_clip = (view_ymin, view_ymax)` 와 정확히 일치.
- spec y범위 + 헤드룸 7%가 view_ymin/ymax. 곡선이 그 밖으로 나가면 segment 끊음.
- ❌ 위반: y_clip을 더 크게 잡으면 곡선이 viewBox 밖으로 나가 잘림 (v3 이전)

**clipPath**: viewBox 전체. 좌표축 박스로 제한하지 않음 (그러면 화살촉 가려짐).

## 규칙 4. 함수 라벨 자동 배치 (`y=...` 류)

spec의 `라벨` 필드 텍스트가 정규식 `^\s*\$?\s*(y\s*=|f\s*\(x\)\s*=)` 매치하면 **함수 라벨**로 인식.

**규정**:
1. spec에서 받은 (lx, ly) 좌표 **무시**
2. 같은 인덱스의 `함수` (없으면 첫 함수)의 visible 영역 75% 지점이 anchor
3. `lx = anchor_x + (xmax-xmin) × 0.04`
4. `func_at_lx = eval(expr, x=lx)` 다시 계산
5. `ly = func_at_lx + (ymax-ymin) × 0.08` (곡선에서 위로 8% 띄움)

**❌ 위반 사례**: y=x 라벨이 직선 위에 정확히 얹힘 (v3 이전 — anchor_y에 +offset만 줘서 기울기 1인 함수에서 무력화)

**코드 위치**: `graph_service._render_svg` 의 라벨 루프

## 규칙 6. PNG 변환 ★ 100% 1:1 매칭 ★

```python
import re
m = re.search(r'viewBox="0 0 ([\d.]+) ([\d.]+)"', svg)
vb_w, vb_h = int(float(m.group(1))), int(float(m.group(2)))
png_bytes = svg_to_png(svg, width=vb_w, height=vb_h)
```

**❌ 위반**: `svg_to_png(svg, 640, 480)` 같이 viewBox 크기와 다르게 전달 — 비례 왜곡 또는 라벨 흐릿
**✅ 정상**: viewBox 크기 == PNG 픽셀 크기. 확대/축소 없음.

**코드 위치**: `graph_service._process_graphs_svg_to_png` + 주공 직접 호출 시 동일

## 규칙 7. hwpx 임베딩 ★ 절대 규정 ★

### 7-1. 세 곳 동시 갱신

| 위치 | 내용 |
|---|---|
| `BinData/graph{N}.png` | PNG 바이트 (zip 안 추가) |
| `Contents/content.hpf` | `<opf:item id="graph{N}" href="BinData/graph{N}.png" media-type="image/png" isEmbeded="1"/>` |
| `Contents/section0.xml` | `[GRAPH:N]` 단독 라인을 `<hp:p id="...">` picture paragraph 로 치환 |

### 7-2. picture paragraph 정확한 형식 (한글 거부 방지)

```xml
<hp:p id="..." paraPrIDRef="0" styleIDRef="0" pageBreak="0" columnBreak="0" merged="0">
  <hp:run charPrIDRef="0">
    <hp:pic id="..." instid="..." ...>...</hp:pic>  ← hp:ctrl 안 감쌈!
    <hp:t/>                                          ← 빈 텍스트 노드 필수
  </hp:run>
  <hp:linesegarray>                                  ← 누락 시 한글 거부
    <hp:lineseg textpos="0" vertpos="0" vertsize="..." ... flags="393216"/>
  </hp:linesegarray>
</hp:p>
```

**❌ 위반 사례**:
- `<hp:run><hp:ctrl><hp:pic>...</hp:pic></hp:ctrl></hp:run>` (v1 — hp:ctrl 잘못 감싸 한글 거부)
- `<hp:t/>` 누락 (v1)
- `<hp:linesegarray>` 누락 (v1)

### 7-3. 단위 변환

- 1px ≒ **26.46 unit** (96 dpi 기준 HWP 1/100mm)
- MAX_W = **16000 unit** (≒160mm). 더 크면 비례 축소
- orgSz = curSz (100% 사이즈)

**코드 위치**: `hwpx_service._build_picture_paragraph`, `_build_picture_xml`, `_inject_graph_manifest`

## 규칙 8. 해설 표준 ★ 새 규정 (2026-04-25 주공 지시) ★

### 8-1. 절대 금지

> "그래프에서 …임을 알 수 있다" 식의 **그림 정보를 답으로 직접 읽는** 해설.
> 학생한테 그림 보고 때려 맞추라는 식이라 교육과정 위반.

### 8-2. 정통 풀이 흐름

이차함수가 그림으로 주어진 경우:
1. **미지수 도입** — 최고차항 계수를 `a (a≠0)`로 놓는다
2. **조건 1**: 그래프의 x절편(α, β)에서 `f(x) = a(x-α)(x-β)`
3. **조건 2**: 그래프의 y절편 또는 다른 정보로 `f(0)` 또는 `f(c)` 값을 알 수 있다
4. **a 결정**: 조건 2 식으로 a를 구함
5. **f(x) 결정** → 대입 → 답

### 8-3. 다른 케이스도 동일 패턴

| 그래프 | 미지수 | 결정 조건 |
|---|---|---|
| 이차함수 | `a (a≠0)` | x절편 + y절편 |
| 일차함수 | `m (m≠0)`, `n` | 두 점 |
| 삼각함수 | 진폭·주기 | 최댓값·주기 |
| 지수함수 | `a (a>0, a≠1)` | 한 점 |

**코드 위치**: 백엔드 코드 아닌 `backend/prompts/solve_prompt.txt` 시스템 프롬프트 — 추후 보강 필요 (현재 갈량의 테스트 spec에서만 정통 풀이 사용)

## 규칙 9. 자동 검증 체크리스트

새 그래프 hwpx 생성 시 자동 검증:

```python
import zipfile, io, re
def verify_hwpx_with_graph(hwpx_bytes: bytes, expected_graph_count: int) -> dict:
    with zipfile.ZipFile(io.BytesIO(hwpx_bytes), 'r') as z:
        bin_graphs = [n for n in z.namelist() if n.startswith('BinData/graph')]
        assert len(bin_graphs) == expected_graph_count, "BinData 누락"

        hpf = z.read('Contents/content.hpf').decode('utf-8')
        for i in range(expected_graph_count):
            assert f'graph{i}' in hpf, f"content.hpf 에 graph{i} 누락"

        section = z.read('Contents/section0.xml').decode('utf-8')
        assert '[GRAPH:' not in section, "잔여 [GRAPH:N] 텍스트"
        for i in range(expected_graph_count):
            assert f'binaryItemIDRef="graph{i}"' in section, f"hp:pic graph{i} 참조 누락"
        assert '<hp:linesegarray' in section, "linesegarray 누락 (한글 거부 위험)"
        assert '<hp:ctrl><hp:pic' not in section, "hp:ctrl 안 hp:pic 잘못된 패턴"
    return {"ok": True, "graph_count": expected_graph_count}
```

**위반 발견 시 즉시 코드 수정 + 다시 검증 + 다시 한글에서 시각 확인**.
