# 유사문제 찾기 GUI 앱 설계

**작성일**: 2026-04-21
**상태**: 설계 완료

---

## 목적

선생님이 기준 문제 1개(HWPX)와 문제집(HWPX, 수백 개)을 입력하면, 문제집 안에서 기준 문제와 유사한 문제의 **번호**를 찾아 알려주는 로컬 GUI 앱.

**핵심 요구사항:**
- **우선순위 1**: 쌍둥이급(숫자·변수만 변형된 수준) 탐지
- **우선순위 2**: 같은 유형/개념/난이도의 유사 문제 탐지
- 로컬 실행(웹앱 미포함), 선생님 개인 사용
- 속도와 정확도 모두 중요

## 아키텍처

### 파일 구조

```
MathSolution/
├── tools/
│   └── similarity_finder/
│       ├── main.py          # Tkinter GUI 진입점
│       ├── comparator.py    # Claude API 호출 + 프롬프트 빌드
│       ├── prompt.txt       # 유사도 판정 시스템 프롬프트
│       ├── run.bat          # 더블클릭 실행용
│       └── logs/            # 요청/응답 로그 (gitignore)
└── backend/services/hwpx_service.py  # 기존 재활용
```

### 의존성

- **Python 표준**: `tkinter`, `threading`, `json`, `pathlib`
- **기존 설치됨**: `anthropic` SDK
- **재활용**: `backend.services.hwpx_service.read_hwpx()`, `split_problems()`
- **설정**: `backend/.env`의 `ANTHROPIC_API_KEY` 재사용

### 데이터 흐름

```
사용자 파일 선택 (원본 HWPX, 문제집 HWPX)
    ↓
hwpx_service.read_hwpx() — 텍스트+수식 추출
    ↓
hwpx_service.split_problems() — 번호별 분할 (문제집)
    ↓
comparator.compare(원본, 문제들, 모델)
    ↓
Claude API 호출 (Sonnet 4.6 기본 / Opus 4.7 재확인)
    ↓
JSON 파싱 → { 쌍둥이: [...], 유형유사: [...] }
    ↓
GUI 결과창에 번호 + 이유 렌더링
```

## GUI 레이아웃

Tkinter 창, 기본 크기 700x600, 리사이즈 가능.

```
┌─────────────────────────────────────────────────┐
│ 유사문제 찾기                                    │
├─────────────────────────────────────────────────┤
│ 원본 문제 (1문제):                               │
│ [________________________] [파일 선택]           │
│                                                  │
│ 문제집 (여러 문제):                              │
│ [________________________] [파일 선택]           │
│                                                  │
│ 모델: ⦿ Sonnet (기본)  ○ Opus (정밀)            │
│                                                  │
│         [    유사 문제 찾기    ]                │
│                                                  │
│ 진행 상태: ████████░░░░ 문제집 파싱 중...       │
├─────────────────────────────────────────────────┤
│ 결과 (스크롤 가능):                              │
│                                                  │
│ 🎯 쌍둥이급 (숫자만 변형)                        │
│   • 12번 — 같은 이차함수 구조, 계수만 다름      │
│   • 47번 — 동일 문제                             │
│                                                  │
│ 📚 유형 유사                                     │
│   • 5번  — 같은 개념, 난이도 동일               │
│   • 103번 — 같은 개념, 난이도 동일               │
│                                                  │
│ [결과 복사]  [Opus로 재확인]                    │
└─────────────────────────────────────────────────┘
```

### 동작

- **파일 선택**: `filedialog.askopenfilename()`, 필터 `*.hwp;*.hwpx`
- **검색 스레드**: `threading.Thread`로 백그라운드 실행 (GUI 멈춤 방지)
- **진행 단계 메시지**: "파싱 중 → Claude 호출 중 → 결과 정리 중"
- **결과 영역**: `tk.Text` + `Scrollbar`, 등급별 색상 강조
- **결과 복사**: 결과 텍스트를 클립보드에 복사 (`clipboard_append`)
- **Opus 재확인**: 동일 요청을 Opus로 재호출해 결과 덮어쓰기 (캐시 재사용)

## Claude 프롬프트 & I/O

### 시스템 프롬프트 (`prompt.txt`)

```
너는 고등학교 수학 문제 유사도 판정기다.

입력:
- 원본 문제 1개 (선생님이 기준으로 삼는 문제)
- 문제집 N개 (각각 번호가 붙어있음)

판정 기준:
[쌍둥이] — 같은 개념·구조·풀이 방법이며 숫자·변수만 다른 수준. 변형문항 관계.
[유형유사] — 같은 단원·개념·난이도이나 문제 구조가 조금 달라 쌍둥이는 아님.
[관련없음] — 개념이 다르거나 난이도 차이가 큼. 출력에서 제외.

출력은 JSON 한 덩어리만. 다른 텍스트 금지:
{
  "쌍둥이": [{"번호": 12, "이유": "..."}, ...],
  "유형유사": [{"번호": 5, "이유": "..."}, ...]
}

이유는 한 줄 (30자 이내).
```

### 사용자 메시지 포맷

```
# 원본 문제
[원본 HWPX에서 추출한 텍스트+수식 전체]

# 문제집 (총 N문제)
## 1번
[1번 문제 텍스트+수식]

## 2번
[2번 문제 텍스트+수식]
...
```

### API 호출 설정

- `model`: `claude-sonnet-4-6` (기본) / `claude-opus-4-7` (재확인)
- `max_tokens`: 4096
- **Prompt caching**: 시스템 프롬프트 + 문제집 블록에 `cache_control` 적용 → Opus 재확인 시 비용 대폭 감소
- **JSON 파싱 실패 시**: 1회 재시도 ("JSON만 출력하라" 강조)

## 에러 처리 & 엣지 케이스

### 파싱 단계

| 조건 | 처리 |
|------|------|
| HWP(구버전 바이너리) 입력 | "HWPX로 저장 후 다시 시도" 안내 다이얼로그 |
| `split_problems` 결과 0개 | "문제 번호를 감지할 수 없습니다" 다이얼로그 + 중단 |
| 원본에서 2개 이상 감지 | 경고 다이얼로그 → 첫 번째 문제만 사용 (확인 시) |

### Claude 호출 단계

| 조건 | 처리 |
|------|------|
| `ANTHROPIC_API_KEY` 미설정 | 앱 시작 시 감지, 안내 후 종료 |
| 네트워크 오류 | 에러 표시 + 재시도 버튼 |
| 문제집 500개 초과 (컨텍스트 우려) | 자동 100개 단위 배치 분할 후 결과 병합 |
| JSON 파싱 실패 | 로그 저장 경로 안내 + 재시도 1회 자동 |
| Claude가 빈 결과 | "유사한 문제를 찾지 못했습니다" 정상 표시 |

### GUI 단계

| 조건 | 처리 |
|------|------|
| 검색 중 창 닫기 | 확인 다이얼로그 |
| 검색 중 버튼 재클릭 | 버튼 비활성화 |
| 결과 없음 상태에서 복사/재확인 | 해당 버튼 비활성화 |

### 로그

- 파일 경로: `tools/similarity_finder/logs/YYYYMMDD_HHMMSS.log`
- 내용: 요청 프롬프트(요약), Claude 원본 응답, 파싱 결과
- `.gitignore`에 `logs/` 추가

## 성공 기준

- 300문제 문제집 기준 Sonnet으로 **1분 이내** 응답
- 쌍둥이급 탐지 정확도 실사용 테스트에서 선생님이 만족할 수준 (샘플로 검증)
- GUI 멈춤 없이 진행 상태 표시
- 1회 실행 비용 300~700원 (Sonnet 기준)

## 성공 기준 외 / YAGNI

다음은 **하지 않는다**:
- 웹앱 탭 추가
- 여러 원본 문제 동시 검색
- 결과 저장/히스토리
- 다른 OS 배포 (Windows 전용)
- PyInstaller 실행파일 빌드 (개발 환경 `python main.py`로 충분)
