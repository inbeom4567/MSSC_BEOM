# Design: 그래프 정확도 고도화 — 실제 기출 이미지 기반 SVG 규칙 추출

## Problem Statement

그래프의 핵심 목적: **해설의 흐름과 맞는 그림을 적절하고 정확하게**. 문제에 들어가든 해설에 들어가든 수능 형식과 최대한 비슷하게.

두 가지 차원의 정확도가 모두 필요:

1. **수학적 정확도**: 해설 흐름에 맞게 올바른 함수/점/구간이 표현됨 (풀이 논리와 일치)
2. **시각적 정확도**: 수능/교과서 SVG 스타일과 일치 (축 화살표, 눈금, 라벨 위치 등)

현재 증상:

- 축 화살표 방향/크기가 다름
- 눈금 위치, O 원점 표기 스타일 불일치
- 함수 곡선의 굵기/정확도
- 점근선, 교점, 라벨 위치가 수능 지면과 다름
- 여러 문제가 복합적으로 발생

현재 방식: solve_prompt.txt에 텍스트로 SVG 규칙을 설명. graph_style_profile.json 존재하지만 SVG 생성 프롬프트에 직접 주입되지 않음.

## Root Cause

실제 기출 시험지의 그래프가 어떻게 생겼는지 "본 적 없는 상태"에서 텍스트로만 규칙을 설명하는 것. 수능 그래프는 매우 특정한 시각적 컨벤션을 가짐.

## Selected Approach: 단일 분석 스크립트

`backend/scripts/analyze_book_graphs.py` 하나로 전체 파이프라인 처리.

### 파이프라인 흐름

```text
book/**/*.hwpx
  → ZipFile로 BinData/* 이미지 추출 (PNG/BMP/JPG)
  → 기존 analyze_graph()로 그래프 여부 필터링 (has_graph == true)
  → 신규 analyze_graph_style()로 시각 규칙 추출
  → backend/data/graph_style_report.json 저장
  → (수동) 검토 후 solve_prompt.txt SVG 섹션 업데이트
```

### 컴포넌트 상세

**1. `backend/scripts/analyze_book_graphs.py`**

- `book/` 하위 모든 `.hwpx` 파일 재귀 탐색
- 각 파일을 ZipFile로 열어 `BinData/` 안의 이미지 파일 추출
- base64 인코딩 후 `analyze_graph()` 호출 → `has_graph=True`만 통과
- 통과한 이미지에 `analyze_graph_style()` 호출
- 결과 집계 → `backend/data/graph_style_report.json` 저장
- 진행 상황 콘솔 출력 (처리 파일 수, 그래프 수, 에러)

**2. `gemini_service.py` — `analyze_graph_style()` 추가**

기존 `analyze_graph()`는 함수식/좌표 추출용. 신규 함수는 **시각 스타일** 추출:

```python
def analyze_graph_style(image_base64: str, media_type: str) -> dict:
    """수능/교과서 그래프 이미지에서 SVG 렌더링에 필요한 시각 스타일 규칙 추출.

    Returns:
        {
          "axis_arrow": str,       # 화살표 모양/크기 설명
          "tick_marks": str,       # 눈금 스타일 (길이, 위치)
          "origin_label": str,     # O 원점 표기 위치/크기
          "curve_style": str,      # 곡선 선 굵기/색상
          "asymptote_style": str,  # 점근선 스타일 (dashed 등)
          "point_style": str,      # 좌표점 원 크기/채우기
          "label_placement": str,  # 함수명/좌표 라벨 위치 패턴
          "shading_style": str,    # 음영 처리 방식
          "overall_size": str,     # 그래프 전체 비율/여백
          "svg_notes": str,        # SVG 코드 작성 시 주의사항
        }
    """
```

**3. `backend/data/graph_style_report.json` (출력)**

```json
{
  "analyzed_at": "2026-04-18T...",
  "total_images": 230,
  "graph_images": 67,
  "styles": [
    {"file": "2025년 고3 수능.hwpx", "image": "BinData/img1.png", "style": {...}},
    ...
  ],
  "aggregated": {
    "axis_arrow": "가장 많이 관찰된 패턴...",
    ...
  }
}
```

**4. solve_prompt.txt SVG 섹션 업데이트 (수동)**

`graph_style_report.json` 검토 후 `## 그래프 출력 규칙 (SVG 방식)` 섹션에 구체적 수치 반영.

| `backend/scripts/analyze_book_graphs.py` | 신규 생성 |

## Constraints

- Gemini API 비용: 100장 × ~0.1원 = 10원 수준
- 스크립트는 일회성 실행 (프로덕션 미포함)
- book/ 파일 접근 경로: 프로젝트 루트 기준 상대 경로

## Success Criteria

- 그래프 이미지 50장 이상 분석 완료
- `graph_style_report.json`에 집계된 공통 패턴 확인
- solve_prompt.txt SVG 섹션에 수치 기반 규칙 5개 이상 추가
- 이전 대비 생성 SVG가 수능 스타일에 더 가까워짐 (육안 확인)
