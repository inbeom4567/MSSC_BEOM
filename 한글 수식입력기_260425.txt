0. 본문 작성 원칙 (가장 중요)
────────────────────────────────────────────────────────────────

[0-1] 모든 숫자·알파벳(변수)은 대괄호 [ ]로 감싼다 — 예외 없음

| 잘못된 예시              | 올바른 예시                |
|--------------------------|----------------------------|
| 함수 f의 개수는 512      | 함수 [f]의 개수는 [512]    |
| ① 840                   | ① [840]                   |
| n은 자연수               | [n]은 자연수               |
| 점 P(a, b)               | 점 [rmP it left(a,~b right)] |

[0-2] 쉼표로 나열된 숫자·변수의 처리 방식

원칙 : **문장의 의미·맥락에 따라 그때그때 다르다.**
       아래 두 패턴 (A)·(B)를 구분해서 적용한다.

──────────────────────────────────────────────────────────────
(A) 분리 — 일반 나열, 서로 다른 수가 "있다/이다" 류 서술
  · 경우의 수 문제에서 서로 다른 수가 존재한다고 서술할 때
    ○ 서로 다른 수 [1], [2], [3], [4], [5]가 있다.
    ○ 자연수 [1], [2], [3]이 차례로 주어진다.
  · 일반 문장에서 변수·값의 병렬
    ○ [a], [b]가 존재한다
    ○ [f left(1 right) = 3], [f left(2 right) = 5]
  · 좌표·구간 등 독립된 수식들의 병렬
    ○ 점 [left(1,~2 right)], [left(3,~4 right)]
    ○ 구간 [left[`0,~1`right]], [left[`2,~3`right]]에서

──────────────────────────────────────────────────────────────
(B) 한 묶음 — 집합 원소 명시 / 물리적 다발(카드·공) / "중에서 뽑는"
  · 집합의 원소를 명시
    ○ 집합 [left{1,~2,~3,~4,~5 right}]
    ○ [n in left{`3,~4,~5,~6,~7`right}]
  · 카드·공·돌멩이 등 물리적 다발에 적힌 숫자
    ○ 숫자 [1, ~2,~ 3,~ 4,~ 5]가 적힌 카드
    ○ [1, ~2,~ 3,~ 4,~ 5]가 적힌 공이 들어 있는 주머니
  · "~ 중에서 뽑는다 / ~ 중에서 고른다" 류 — 다발에서 추출
    ○ [1, ~2,~ 3,~ 4] 중에서 서로 다른 [4]개를 택한다

  ※ 묶을 때는 쉼표 뒤에 `~`(틸데, 큰 공백)을 넣어 가독성 보장.
    예 : `1, ~2,~ 3,~ 4`

──────────────────────────────────────────────────────────────
구분 요약
  · "있다 / 이다" 식 단순 술어  → (A) 분리
  · "적힌 / 새겨진 / 라벨된"     → (B) 한 묶음
  · "집합 / 원소 / { … }"         → (B) 한 묶음
  · "중에서 ~ 개를 뽑는다"         → (B) 한 묶음

[0-3] 모든 수학 기호·수식·상수도 대괄호 안에

  × π는 자연수가 아니다           ○ [pi]는 자연수가 아니다
  × x ≤ 3 인 모든 정수            ○ [x le 3]인 모든 정수
  × 0 < b-a ≤ n/2                 ○ [0 < b-a le {n} over {2}]

1. 수식 표기 절대 규칙 9가지
────────────────────────────────────────────────────────────────

[규칙 1] 모든 수학 요소는 [ ] 안에
  × 자연수 n의 값의 합을 구하시오.
  ○ 자연수 [n]의 값의 합을 구하시오.

  × f(x) = 2x² + 3x - 1
  ○ [f left(x right) ] [= 2x^{2}+3x-1]

[규칙 2] 괄호는 반드시 left( right) — 일반 ( ) 금지
  × [(a+b)^{2}]                    ○ [left( a+b right)^{2}]
  × [f(x) = log_{3}(x+1)]
  ○ [f left(x right) ] [= log_{3}`{left( x+1 right)}]
  × [P(A∩B)]                       ○ [rmP it left(A cap B` right)]

[규칙 3] ± 는 반드시 +- (pm 금지)
  × [x = {-b pm sqrt{b^{2}-4ac}} over {2a}]
  ○ [x ] [= {-b +- sqrt{b^{2}-4ac}} over {2a}]

[규칙 4] 수식은 한 줄에 하나, 세로 배치
  하나의 [ ] 안에 여러 등식·부등식을 나열하지 않는다.
  × [a=b LRARROW c=d]
  ○ [a=b]
    이므로
    [c=d]
  × [x^{2}=4,~ x=+-2]
  ○ [x^{2}=4]
    [therefore~ x = +- 2]

  예외 — 연립방정식·범위 표현(쉼표 구분)은 한 줄 허용
  ○ [0 < a < n,~ 0 < b < n]

[규칙 4-1] 등호가 있는 식은 등호 앞에서 [ ] 분리
  × [alpha^{7}+beta^{7} = left(alpha^{4}+beta^{4} right)left(alpha^{3}+beta^{3} right)]
  ○ [alpha^{7}+beta^{7} ] [= left(alpha^{4}+beta^{4} right)left(alpha^{3}+beta^{3} right)]

  × [f left(x right) = 2x^{2}+3x-1]
  ○ [f left(x right) ] [= 2x^{2}+3x-1]

  예외 — 부등식·조건식은 분리 금지
  ○ [0 < a < n]    ○ [x le 3]

[규칙 5] 어림값 계산 금지
  × [sqrt{12} approx 3.46]                ○ [sqrt{12} = 2 sqrt{3}]
  × [sqrt{3} < 2]   (이유: 1.73... < 2)
  ○ [sqrt{3} < 2]   (이유: [left( sqrt{3} right)^{2} = 3 < 4 = 2^{2}])

  유일한 예외 : 문제에서 "단, log 2 = 0.3"처럼 명시한 경우.

[규칙 6] 쉼표 뒤 반드시 띄어쓰기
  × [3],[6],[9]            ○ [3], [6], [9]
  × [a],[b]가 존재한다     ○ [a], [b]가 존재한다

[규칙 7] 마침표·물음표는 [ ] 밖에
  × [3+6+9 = 18.]                ○ [3+6+9 ] [= 18]
  × [n]의 값을 구하시오[.]       ○ [n]의 값을 구하시오.

[규칙 8] 수식 내부에 한글 금지
  × [x는 자연수]                       ○ [x]는 자연수
  × [f left(x right) 는 이차함수]      ○ [f left(x right)]는 이차함수

[규칙 9] 조건 번호(①②③, ㄱㄴㄷ 등)는 [ ] 밖에
  × [① f left(1 right) = 3]      ○ ① [f left(1 right) = 3]
  × [ㄱ. a > 0]                   ○ ㄱ. [a > 0]

────────────────────────────────────────────────────────────────

[2-1] 한글 → 수식 : 반드시 띄어쓰기
  × 자연수[n]은              ○ 자연수 [n]은
  × 함수[f left(x right)]가  ○ 함수 [f left(x right)]가

[2-2] 수식 → 조사 : 붙이기
  ○ [n]은, [a]와, [3]이, [x]를, [f left(x right)]의

[2-3] 수식 → 수식 : 사이에 한글이 없으면 쉼표+띄어쓰기
  ○ [a], [b]가 존재한다
  ○ [f left(1 right) = 3], [f left(2 right) = 5]
  ○ 점 [left(1,~2 right)], [left(3,~4 right)]를
  ○ 구간 [left[`0,~1`right]], [left[`2,~3`right]]에서

[2-4] 합성어(붙여쓰기) — 수학 용어는 한글 숫자 사용

  ○ 이차부등식, 삼각함수, 사분면, 이항정리

  단, 변수일 경우는 [ ] 사용
  ○ [n]차 다항함수   ([n] 이 변수)

3. 객관식 보기 표기 (신규 — 2026-04-25)
────────────────────────────────────────────────────────────────

객관식 5지선다 보기를 한 줄에 나열할 때, 보기 사이 구분자는
**스페이스 공백 금지, 탭(Tab) 2번**으로 통일한다.

이유 : 한글에서 스페이스로 보기 간격을 맞추면 폭이 들쭉날쭉
하지만, 탭 2번을 일관 적용하면 모든 보기가 같은 간격으로
자동 정렬되어 가독성이 보장된다.

[3-1] 표기 형식
  ① [보기1](탭)(탭)② [보기2](탭)(탭)③ [보기3](탭)(탭)④ [보기4](탭)(탭)⑤ [보기5]

[3-2] 예시 (한 줄 가로 배치)
  ① [3]		② [4]		③ [5]		④ [6]		⑤ [7]
  (각 보기 사이는 실제 키보드 Tab 키 두 번 — 본 문서에서 위
   공백은 탭 두 번이 들어간 자리이다)

[3-3] 보기 5개 모두 수식인 경우도 동일
  ① [{1} over {2}]		② [{1} over {3}]		③ [{1} over {4}]		④ [{1} over {5}]		⑤ [{1} over {6}]

[3-4] 가나다 보기형(ㄱ, ㄴ, ㄷ만 있는 경우)
  ① ㄱ		② ㄴ		③ ㄱ, ㄴ		④ ㄴ, ㄷ		⑤ ㄱ, ㄴ, ㄷ

[3-5] 절대 금지
  × ① [3]  ② [4]  ③ [5]  ④ [6]  ⑤ [7]    (스페이스 사용)
  × ① [3] ② [4] ③ [5] ④ [6] ⑤ [7]        (한 칸 띄어쓰기)
  × ①[3] ②[4] ③[5] ④[6] ⑤[7]             (붙여쓰기)

[4-6] 함수 정의역 → 공역 표기 (집합 간 함수)
  f : X → Y             f`:` X ~->~ Y
  f : A → B (집합기호)  f`:` A ~->~ B
  f : {1,2,3} → {1,2,3,4,5}
                        f`:` left{1,~2,~3 right} ~->~ left{1,~2,~3,~4,~5 right}
  f : ℝ → ℝ             f`:` {bf R} ~->~ {bf R}
  (g ∘ f) : X → Z       left(g circ f right)`:` X ~->~ Z

  ※ 절대 금지 (LaTeX 백슬래시 형식)
    × f : \{1, 2, 3\} \to \{1, 2, 3, 4\}
    × f\colon \{1,2,3\} \rightarrow \{1,2,3,4\}
    × f: \{a, b\} \to \{c, d\}
    ○ f`:` left{1,~2,~3 right} ~->~ left{1,~2,~3,~4 right}

  ※ 핵심 3 규칙
    1) 중괄호 집합 {…} 은 반드시 left{ … right} (LaTeX의 \{ 금지)
    2) 화살표 → 는 -> 이며 양쪽에 ~ 한 칸 (~->~)
    3) 콜론 : 은 함수 표기에서 백틱으로 감싸기  f`:`
       (일반 문장 내 콜론과 구분)

  ※ 원소 나열 시 쉼표 뒤 ~ 추가 (1,~2,~3) — 마지막 뒤에는 ~ 불필요

[4-11] 확률 · 통계 — 순열 / 조합 / 중복조합 / 중복순열

  ── 순열 (P) ──

  ₙCᵣ (중간)            _{n}{rmC it}_{r}
  ₙCᵣ (수식 맨 앞)      {}_{n}{rmC it}_{r}

  ── 중복조합 (H) ──
  ₙHᵣ (중간)            _{n}{rmH it}_{r}
  ₙHᵣ (수식 맨 앞)      {}_{n}{rmH it}_{r}

  ── 중복순열 (Π) ── ★ smallprod 사용 (rm 없음) ★
  ₙΠᵣ (중간)            _{n} smallprod _{r}
  ₙΠᵣ (수식 맨 앞)      {}_{n} smallprod _{r}

  예시
    [{}_{6}{rmP it}_{3} = 120]
    [{}_{6}{rmC it}_{2} = 15]
    [{}_{4} smallprod _{4} - _{3} smallprod _{4} = 256-81 = 175]

  ── 확률 (rm 로마체 P, 뒤에 백틱) ──
  P(A)                  rmP it left(A` right)
  P(A∩B)                rmP it left(A cap B` right)
  P(A|B)                rmP it left(A`left|`B`right)   (조건부 확률)
  P(Aᶜ) (여사건)        rmP it left(A^{C} right)

  ── 통계 ──
  E(X)                  rmE it left(X` right)
  V(X)                  rmV it left(X`right)
  N(m, σ²)              rmN it left(m,~sigma^{2} right)   (정규분포)
  P(Z ≤ z)              rmP it left(Z le z right)
  x̄ (평균)              {bar{X`}}
  σ²                    sigma^{2}
  σ                     sigma

[4-12] 기타 기호
  π                     pi
  ∞                     inf
  ∴                     therefore~              (뒤에 ~ 권장)
  ∵                     because~
  →                     ->                      (또는 rarrow)
  ≡ (합동)              ==
  ∼ (닮음)              sim
  작은 공백             `                       (백틱 1개)
  큰 공백               ~                       (백틱 4개와 동일)

[4-13] 조각 함수 cases (구간별 정의 함수)
  조각 함수는 반드시 cases{ ... } 문법 사용. matrix 사용 금지.

  형식
    cases{ `식1 && 조건1# `식2 && 조건2# `식3 && 조건3 }

  · 각 행 :  `식 && 조건   — 식 앞에 반드시 백틱(`)
  · 행 구분 :  #          (# 앞에 공백 없음.  예: right)#)
  · &&  : 오른쪽 정렬 조건 의미
  · LaTeX의 \begin{cases}…\end{cases} 는 반드시 cases{ … } 로 변환
  · matrix{n}{m} 사용 절대 금지

  예시 1 — 두 구간
    f(x) = x²+2x+1 (x≤1), x (x>1)
    [f left(x right) = cases{ ` x^{2}+2x+1 && left( x le 1 right)# `x && left( x>1 right) }]

  예시 2 — 세 구간
    [f left(x right) = cases{ `x && left( x < -1 right)# `f left(x right) && left( -1 le x le 1 right)# `2-x && left( x > 1 right) }]

  예시 3 — 한글 조건 (홀짝)
    [g left(n right) = cases{ `0 && left( n이 짝수 right)# `1 && left( n이 홀수 right) }]

[4-14] 폰트 전환 규칙
  상황                  코드          설명
  ─────────────────────────────────────────────────────────────
  기본                  (없음)        이탤릭체가 기본
  로마체로 전환         rm            도형 점, P, E, V, N, C, H 등
  이탤릭체로 복귀       it            rm 이후 다시 이탤릭 필요 시
  로마 대문자+괄호      백틱(`) 추가  rmP it left(A` right)
  수식 맨 앞 아래첨자   {} 선행       {}_{n}{rmC it}_{r}
  ─────────────────────────────────────────────────────────────

  ※ 도형의 점(A,B,C 등), 확률 P, 기댓값 E, 분산 V, 정규분포 N,
    순열 P, 조합 C, 중복조합 H 등은 반드시 rm 사용.
  ※ 단, 중복순열 Π(파이)는 smallprod 로 입력하며 rm·it 불필요.
  ※ rm 다음 식이 이어질 때는 it 으로 이탤릭 복귀:
    × [rm{bar{AB}} = k^{2} - left(-2 right) = k^{2} + 2]
       (k 가 로마체로 잠겨버림)
    ○ [rm{bar{AB}} = it k^{2} - left(-2 right) = k^{2} + 2]
  ※ 로마체 대문자 알파벳으로 수식이 끝나거나 괄호가 들어가는
    경우 백틱 입력 :  rmP it left(A` right)

5. 자주 발생하는 수학적 오류 방지
────────────────────────────────────────────────────────────────

[5-1] 판별식
  · 일차항 계수 홀수 → [D = b^{2}-4ac]
  · 일차항 계수 짝수 → [D/4 = left({b} over {2} right)^{2}-ac]
  · D = 0  : 접한다 (중근)
  · D > 0  : 서로 다른 두 실근
  · "서로 다른 두 양의 실근" → 합 > 0, 곱 > 0 추가 확인

[5-2] 로그
  · 진수 > 0,  밑 > 0,  밑 ≠ 1 반드시 확인
  · 밑 > 1 → 증가함수 / 0 < 밑 < 1 → 감소함수
  · 밑 변환 :  [log_{a}`{b} = {log_{c}`{b}} over {log_{c}`{a}}]
  · [log_{a}`{b} cdot log_{b}`{a} = 1]

[5-3] 부등식 · 범위
  · 등호 포함 여부 주의 (≤ 와 < 는 결과가 다를 수 있음)
  · 이하/이상 → 등호 포함 (≤, ≥)
  · 미만/초과 → 등호 미포함 (<, >)
  · 절댓값 부등식 :  [left|`x`right| < a LRARROW -a < x < a]
  · 이차부등식의 해 : 판별식 부호에 따라 전체 실수/공집합 가능

[5-4] 원과 직선
  · 원 위의 점 : 원 방정식에 대입하여 등호 성립 확인
  · 두 원의 위치 관계 (중심거리 d, 반지름 r₁, r₂)
      외접     : [d = r_{1}+r_{2}]                     교점 1개
      내접     : [d = left|`r_{1}-r_{2}`right|]        교점 1개
      두 점 교차: [left|`r_{1}-r_{2}`right| < d < r_{1}+r_{2}]
                                                       교점 2개
  · 대소비교는 양변 제곱 등 정확한 방법

[5-5] 삼각함수
  ① 호도법 ↔ 도수법
    π/6 = 30°    π/4 = 45°    π/3 = 60°    π/2 = 90°
    2π/3 = 120°  3π/4 = 135°  5π/6 = 150°  π = 180°
    7π/6 = 210°  5π/4 = 225°  4π/3 = 240°  3π/2 = 270°
    5π/3 = 300°  7π/4 = 315°  11π/6 = 330° 2π = 360°
  ② 사분면별 부호
    1사분면 (0°~ 90°)  sin +  cos +  tan +
    2사분면 (90°~180°) sin +  cos -  tan -
    3사분면 (180°~270°)sin -  cos -  tan +
    4사분면 (270°~360°)sin -  cos +  tan -
  ③ 특수각 삼각비
        0°    30°       45°      60°      90°
   sin   0    1/2       √2/2     √3/2      1
   cos   1    √3/2      √2/2     1/2       0
   tan   0    √3/3       1       √3       없음

[5-6] 수열 · 급수
  · 등차수열 :  [a_{n} = a_{1}+left(n-1 right)d]
  · 등비수열 :  [a_{n} = a_{1} cdot r^{n-1}]
  · 등차합   :  [S_{n} = {n left(a_{1}+a_{n} right)} over {2}]
  · 등비합 (r≠1) : [S_{n} = {a_{1} left(1-r^{n} right)} over {1-r}]
  · [a_{n} = S_{n}-S_{n-1}]  (n ≥ 2),  [a_{1} = S_{1}] 별도 확인 필수

[5-7] 미적분
  · 미분 :  [left(x^{n} right)' = nx^{n-1}]
  · 적분 :  [int x^{n}`dx = {x^{n+1}} over {n+1} + rmC it]   (n ≠ -1)
  · 곱의 미분 :  [left(fg right)' = f'g+fg']
  · 합성함수 미분 (겉미분 × 속미분)
       [left(f left(g left(x right) right) right)' = f' left(g left(x right) right) cdot g' left(x right)]
  · 정적분과 넓이 :  넓이 = [int_{a}^{b} left|`f left(x right)`right|`dx]
                                                      (절댓값 주의!)

[5-8] 계산 복잡도 제한
  · 지수 계산 결과는 5000 이하 (2¹¹=2048, 3⁵=243 정도까지)
  · 어림값 사용 금지 (문제에서 명시하지 않는 한)
  · 대소비교는 제곱·로그 등 정확한 방법 사용

6. 글로벌 보조 규칙
────────────────────────────────────────────────────────────────

· font_mode (서체 모드)
    기본은 이탤릭(it). rm 으로 로마체 전환, it 으로 복귀.
    rm 적용 대상 : 도형의 점(A,B,C…), 확률 P, 기댓값 E, 분산 V,
                   정규분포 N, 순열 P, 조합 C, 중복조합 H 등.
    smallprod(중복순열 Π)는 rm·it 불필요.
    예 :  rmP it left(A` right)

· spacing (공백)
    · `  : 작은 띄어쓰기 (강제 공백 1칸)
    · ~  : 큰 띄어쓰기 (백틱 4개와 동일)
    · le, ge, ne 등 키워드는 앞뒤에 다른 문자가 오면 공백 필요
      예 :  x ge 3

· brackets (괄호)
    · 일반 괄호 : left( … right)         — 함수명과 left 사이 공백
    · 중괄호    : left{ … right}         — 코드의 {} 와 구분
    · 대괄호    : left[` … `right]
    · 절댓값    : left|` … `right|

· 수식 맨 앞 아래첨자 : 빈 중괄호 선행
    예 :  {}_{n}{rmC it}_{r}

· 로마체 대문자 + 괄호 : 백틱 추가
    예 :  rmP it left(A` right)

· 수식 내 줄바꿈 #
    구간별 정의 함수(cases)에서만 사용. 그 외에는 별도 [ ] 로 분리.

· 수식 내 한글
    원칙적으로 수식 내부에 한글 금지.
    꼭 필요하면 수식창과 한글창을 분리하여 입력.

· 객관식 보기 구분
    스페이스 금지. 탭(Tab) 2번으로 일정 간격 확보 (3장 참조).

[보강 1] 그리스 문자 ω(오메가)는 반드시 omega ★

  · ω 입력 시 영문 알파벳 'w'는 절대 금지
    × [w^{3} =1]                    (영문 w로 인식되어 잘못 출력)
    ○ [omega^{3} =1]
    ○ [1+omega+omega^{2} =0]

  · 다른 그리스 문자도 동일 — 영문 알파벳 모양만 비슷해도 절대 금지
    α=alpha, β=beta, γ=gamma, δ=delta, ε=epsilon, θ=theta,
    λ=lambda, μ=mu, π=pi, ρ=rho, σ=sigma, τ=tau,
    φ=phi, χ=chi, ψ=psi, ω=omega
  · 대문자: 첫 글자 대문자 (Alpha, Theta, Pi, Sigma, Phi, Psi, Omega 등)
    단, 합기호 Σ는 sum, 곱기호 Π는 prod 사용 (이전과 동일)


[보강 2] 사용자 정의 이항연산 별표는 유니코드 ∗ (U+2217) ★

  · 키보드 별표 `*` (U+002A) 사용 금지 — 키보드 위쪽 작은 별표로 인식됨
  · 명령어 `ast` 등도 사용 금지
  · 반드시 유니코드 ∗ (U+2217) 직접 입력
    × [A*B = 2A-B^{2}]              (키보드 별표 — 잘못된 출력)
    × [A ast B = 2A-B^{2}]          (명령어 — 잘못된 출력)
    ○ [A∗B = 2A-B^{2}]              (유니코드 ∗ 직접 입력)
  · 예시 (등호 분리 규칙 적용)
    [left(x^{2} -xy+2y^{2} right) ∗ left(2x-y right)]
    [=2 left(x^{2} -xy+2y^{2} right) -left(2x-y right) ^{2}]
    [=-2x^{2} +2xy+3y^{2}]
  · 복사용 별표 한 글자: ∗
