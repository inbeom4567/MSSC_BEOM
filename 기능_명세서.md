## 1. 프로젝트 개요

**MathSolution**은 고등학교 수학 교사를 위한 AI 기반 유사문항 생성 도구입니다.

### 핵심 목적

- 수학 문제/해설 이미지 또는 HWPX 파일을 입력받아 유사문항과 풀이를 자동 생성
- 출력 형식: 한글 수식입력기(HWP) 코드 — 대괄호 `[수식코드]` 형태
- 생성된 내용을 바로 한글 파일(.hwpx)로 다운로드 가능

### 지원 입력 형태

| 입력 | 설명 |
|------|------|
| PNG/JPG 이미지 | 문제 및 해설 이미지 (스캔본, 사진 등) |
| PDF | 스캔 처리 탭에서 지원, 페이지별 이미지로 변환 |
| HWPX 파일 | 한글 문서 파일 직접 파싱 |

### 지원 출력 형태

| 출력 | 설명 |
|------|------|
| HWP 코드 텍스트 | 대괄호 `[수식]` 형태, 클릭하면 클립보드 복사 |
| 그래프 이미지 | Matplotlib으로 생성된 PNG (함수 그래프, 기하) |
| HWPX 파일 | 결과가 담긴 한글 문서 파일 다운로드 |

## 2. 기술 스택 및 아키텍처

### 전체 아키텍처

```
사용자 브라우저 (React + Vite :5173)
        ↓  직접 HTTP 요청 (VITE_API_URL=http://localhost:8001)
FastAPI 백엔드 (:8001)
        ├── Anthropic Claude API  (claude-sonnet-4, claude-opus-4)
        ├── Google Gemini API     (gemini-2.5-flash)
        └── Matplotlib           (그래프 이미지 생성)
```

**중요**: 프론트엔드는 Vite 프록시를 사용하지 않고 `http://localhost:8001`로 직접 API 호출합니다.

### 기술 스택

| 계층 | 기술 |
|------|------|
| 프론트엔드 | React 19, Vite 8, Tailwind CSS 4 |
| 백엔드 | Python, FastAPI, Uvicorn |
| AI — 문제 생성 | Anthropic Claude API (Sonnet 4 / Opus 4) |
| AI — OCR/이미지 분석 | Google Gemini API (gemini-2.5-flash) |
| 그래프 | Matplotlib (서버사이드 PNG 렌더링) |
| PDF 처리 | PyMuPDF (fitz) |
| 이미지 처리 | Pillow (PIL) |
| 파일 처리 | Python zipfile (HWPX = ZIP + XML) |

### Claude 모델 ID

| 단칭 | 실제 모델 ID | 입력 단가 | 출력 단가 |
|------|------------|---------|---------|
| sonnet | claude-sonnet-4-20250514 | $3/1M | $15/1M |
| opus | claude-opus-4-20250514 | $15/1M | $75/1M |

### Gemini 모델

| 용도 | 모델 |
|------|------|
| 텍스트 분석, OCR | gemini-2.5-flash |
| 이미지 생성 분석 | gemini-2.5-flash-image |

### API 상세

#### `POST /api/generate`

유사문항 생성. 이미지 1~2장 (문제 + 해설) 업로드.

**요청** (multipart/form-data):

| 파라미터 | 타입 | 기본값 | 설명 |
|---------|------|--------|------|
| `files` | File[] | 필수 | 이미지 파일 (문제, 해설 순서) |
| `variant_type` | string | "idea" | "number" (숫자변형) \| "idea" (아이디어변형) |
| `difficulty` | string | "similar" | "easier" \| "similar" \| "harder" |
| `model` | string | "sonnet" | "sonnet" \| "opus" |
| `custom_prompt` | string | "" | 추가 지시사항 |
| `grade` | string | "none" | 학년 코드 (예: "high2_math1") |

**응답** (JSON):

```json
{
  "result": "-유사문항-\n...\n-정답-\n...\n-해설-\n...",
  "graphs": [
    {"index": 0, "image_base64": "...", "media_type": "image/png"}
  ],
  "usage": {
    "input_tokens": 1234,
    "output_tokens": 567,
    "total_tokens": 1801,
    "cost_usd": 0.0234,
    "cost_krw": 34,
    "model": "claude-sonnet-4-20250514"
  },
  "history_id": "abc12345"
}
```

**내부 처리 흐름**:
1. 이미지를 base64로 변환
2. Claude에게 유사문항 생성 요청 (1차 호출)
3. `_cleanup_output()` — 계산 재검산 + 형식 정리 (2차 호출)
4. `process_graphs_in_text()` — `-그래프-` 태그를 Matplotlib PNG로 변환
5. 결과를 히스토리에 저장

---

**요청** (multipart/form-data):

| 파라미터 | 타입 | 기본값 | 설명 |

**응답**: `/api/generate`와 동일 구조  
**출력 태그**: `-풀이-`, `-정답-`

#### `POST /api/hwpx-batch`

HWPX 내 선택 문제들을 병렬로 일괄 처리.

**요청** (multipart/form-data):

| 파라미터 | 타입 | 설명 |
|---------|------|------|
| `file` | File | HWPX 파일 |
| `variant_type` | string | "number" \| "idea" |
| `difficulty` | string | "easier" \| "similar" \| "harder" |
| `model` | string | "sonnet" \| "opus" |
| `selected_numbers` | string | "1,2,3" 형식 (빈 문자열=전체) |
| `grade` | string | 학년 코드 |
| `custom_prompt` | string | 추가 지시사항 |

**내부 처리**: `asyncio.gather()` 로 모든 문제를 동시에 Claude API 호출 (병렬 처리)

**응답**:

```json
{
  "results": [
    {"number": 1, "result": "...", "graphs": [], "usage": {...}},
    {"number": 2, "result": "...", "graphs": [], "usage": {...}}
  ],
  "combined_text": "-1번-\n...\n\n-2번-\n...",
  "usage": {"total_tokens": 3456, ...},
  "download_id": "ab12cd34",
  "problem_count": 2,
  "history_id": "xyz"
}
```

#### `POST /api/scan-detect`

이미지 또는 PDF를 업로드하면 Gemini로 각 문제의 위치(bbox)를 감지.

**요청** (multipart/form-data): `file` — 이미지 또는 PDF

**응답**:

```json
{
  "pages": [
    {
      "page_index": 0,
      "image_base64": "...",
      "media_type": "image/png",
      "bboxes": [
        {"id": "p0_b0", "x": 0.05, "y": 0.02, "w": 0.9, "h": 0.3, "label": "문제 1"}
      ]
    }
  ],
  "total_pages": 1
}
```

bbox 좌표는 **이미지 크기 대비 비율** (0.0 ~ 1.0).

---

#### `POST /api/scan-crop-process`

확정된 bbox로 각 문제를 하나씩 처리하며 SSE(Server-Sent Events)로 실시간 결과 스트리밍.

**요청** (JSON):

```json
{
  "pages": [...],
  "confirmed_bboxes": [
    {
      "id": "p0_b0",
      "page_index": 0,
      "x": 0.05, "y": 0.02, "w": 0.9, "h": 0.3,
      "label": "문제 1",
      "selected": true
    }
  ],
  "output_mode": "type_only",
  "variant_count": 1,
  "model": "sonnet",
  "grade": "none"
}
```

**output_mode 옵션**:

| 값 | 설명 |
|----|------|
| `type_only` | 타이핑만 — HWP 코드 변환 (해설/유사문항 없음) |
| `type_with_solution` | 타이핑 + 해설 작성 (유사문항 없음) |
| `variant` | 타이핑 + 해설 + 유사문항 생성 (기본) |

**SSE 이벤트 형식**:

```
data: {"type": "progress", "problem_id": "p0_b0", "label": "문제 1", "status": "processing"}

data: {"type": "result", "problem_id": "p0_b0", "label": "문제 1", "result": "...", "graphs": [], "usage": {...}}

data: {"type": "error", "problem_id": "p0_b1", "label": "문제 2", "error": "오류 메시지"}

data: {"type": "done", "total": 3}
```

**처리 파이프라인** (문제별):
1. bbox 좌표로 이미지 크롭 (`_crop_image()`)
2. Gemini OCR — 수식/텍스트 추출 (`ocr_scan_general()`)
3. Claude 처리 — HWP 변환 + 해설/유사문항 (`process_scan()`)
4. 결과 SSE 전송

---

#### `POST /api/text-to-hwpx`

여러 텍스트 결과를 하나의 HWPX 파일로 합쳐서 다운로드 ID 반환.

**요청** (JSON):

```json
{
  "texts": ["-문제-\n...\n-해설-\n...", "-문제-\n...\n-해설-\n..."],
  "filename": "scan_result"
}
```

---

#### `POST /api/prompt-feedback`

사용자 피드백 텍스트를 Claude로 규칙화하여 프롬프트 파일에 자동 추가.

**요청** (JSON): `{"feedback": "플러스마이너스는 pm이 아니라 +- 로 써야 합니다"}`

**내부 처리**:
1. Claude에게 피드백을 규칙 형태(× 틀림 / ○ 올바름)로 변환 요청
2. `solve_prompt.txt`, `variant_solve_prompt.txt` 모두에 규칙 삽입
3. `claude_service.reload_prompts()` 호출 (서버 재시작 불필요)

---

#### `GET /api/hwpx-download/{download_id}`

임시 저장된 HWPX 파일을 다운로드하고 메모리에서 제거.

- 파일은 메모리에 최대 1시간 저장 (`_hwpx_store`)
- `download_id`는 UUID 앞 8자리

## 5. 탭별 기능 상세

### 탭1 — 유사문항 생성 (TabCreateVariant)

**목적**: 원본 문제/해설 이미지 → 유사문항 + 풀이 자동 생성

**UI 구성**:
- 이미지 드롭존 2개 (원본 문제, 원본 해설) — 드래그앤드롭 또는 클릭 업로드
- 변형 유형 선택: 숫자 변형 / 아이디어 변형
- 난이도 선택: 쉽게 / 비슷하게 / 어렵게
- 추가 지시사항 텍스트 입력 (custom_prompt)
- 생성 버튼 → 로딩 → 결과 표시

**결과 표시** (`SolutionDisplay`):
- `-유사문항-`, `-정답-`, `-해설-` 태그별 섹션 분리 렌더링
- `[수식코드]` → 클릭 가능한 버튼으로 렌더링 (클릭 시 클립보드 복사)
- 그래프 이미지가 있으면 인라인 표시
- 수정 요청 기능: 결과 아래에 지시사항 입력 → `/api/refine` 호출

**API 호출**: `POST /api/generate`

### 탭3 — 스캔 처리 (TabScan)

**목적**: 스캔본 이미지/PDF를 업로드하면 문제를 자동 인식하고 HWP 코드로 변환 + 유사문항/해설 생성

**워크플로우 단계**:

```
[Step 1] 파일 업로드
      ↓
[Step 2] 문제 bbox 자동 감지 (Gemini)
         → 감지된 문제들이 이미지 위에 빨간 박스로 표시
      ↓
[Step 3] bbox 확인/수정
         - 박스 삭제 가능
         - 새 박스 수동 추가 가능
         - 처리 방식 선택: output_mode 선택
      ↓
[Step 4] 문제별 순차 처리 (SSE 스트리밍)
         - 문제별 진행 상황 실시간 표시
         - 각 문제 완료 시 바로 결과 표시
      ↓
[Step 5] 결과 확인 + HWPX 다운로드
```

**처리 방식(output_mode)**:

| 방식 | 설명 |
|------|------|
| 타이핑만 | OCR → HWP 코드 변환만 (해설/유사문항 없음) |
| 타이핑 + 해설 | HWP 변환 + 해설 작성 |
| 유사문항 생성 | HWP 변환 + 해설 + 유사문항 생성 |

**진행 표시**:
- 플로팅 배지: 우하단 고정, "처리 중 2/5" 형태
- 완료된 문제는 결과 카드로 즉시 표시
- 오류 발생 시 해당 문제만 빨간 카드로 표시 (다른 문제 처리는 계속)

**API 호출**:
- Step 2: `POST /api/scan-detect`
- Step 4: `POST /api/scan-crop-process` (SSE)
- 결과 다운로드: `POST /api/text-to-hwpx` → `GET /api/hwpx-download/{id}`

### ClaudeService (`claude_service.py`)

Claude API 호출과 프롬프트 관리를 담당.

#### 초기화 (`__init__`)

```python
self.client = anthropic.AsyncAnthropic(api_key=...)
self.mapping_ref = self._load_mapping_reference()  # hwp_math_mapping.json 로드
self.curriculum = self._load_curriculum()           # curriculum.json 로드
self.solve_prompt = self._build_prompt("solve_prompt.txt")
self.variant_solve_prompt = self._build_prompt("variant_solve_prompt.txt")
```

프롬프트 빌드 시 `mapping_ref` (수식 매핑 사전)와 `fewshot_examples.txt` (있으면)를 자동 결합.

#### 주요 메서드

| 메서드 | 입력 | 설명 |
|--------|------|------|
| `generate_variant(images, variant_type, difficulty, model, custom_prompt, grade)` | 이미지 리스트 | 유사문항 생성 (2회 Claude 호출) |
| `solve_variant(images, model, custom_prompt, grade)` | 이미지 리스트 | 변형문항 풀이 (2회 Claude 호출) |
| `generate_variant_from_text(text, ...)` | HWPX 추출 텍스트 | 텍스트 기반 유사문항 생성 |
| `solve_variant_from_text(text, model, grade)` | HWPX 추출 텍스트 | 텍스트 기반 풀이 생성 |
| `process_scan(ocr_data, mode, variant_count, model, grade, output_mode)` | OCR 결과 dict | 스캔 처리 (HWP 변환 + 해설/유사문항) |
| `refine(original_result, instruction, model)` | 기존 결과 + 지시 | 결과 수정 |
| `process_feedback(feedback)` | 피드백 텍스트 | 규칙 텍스트 생성 |
| `reload_prompts()` | — | 프롬프트 파일 재로드 |

#### 2단계 검증 시스템

모든 생성 작업은 **2회 Claude API 호출**로 수행:

**1차 호출**: 유사문항/풀이 생성  
**2차 호출** (`_cleanup_output` 또는 `_verify_solve_output`):
- 계산 재검산 (처음부터 독립적으로 재계산)
- 문제-풀이 일관성 확인 (숫자/조건 일치)
- 정답 태그 확인
- 내부 사고 과정 제거 (잠깐, STEP, ##, ** 등)
- 수식 대괄호 `[ ]` 형식 유지 확인

#### 학년 제약 프롬프트 (`_get_grade_prompt`)

학년 선택 시 자동으로 시스템 프롬프트에 추가:
```
★★★ 학년 제약: 고2 수I (수학) ★★★
이 학생은 고등학교 2학년 수학I 과정을 배우고 있습니다.
사용 가능한 개념: 지수함수, 로그함수, 삼각함수, 수열
절대 사용 금지 개념: 미분, 적분, 극한, 벡터
위 금지 개념은 아직 배우지 않았으므로 풀이에 절대 사용하지 마세요.
```

### HwpxService (`hwpx_service.py`)

HWPX 파일 읽기/쓰기 담당. HWPX는 ZIP + XML 구조.

#### `read_hwpx(file_bytes) → str`

HWPX에서 텍스트+수식 추출:
1. ZIP 해제 → `Contents/section*.xml` 파일 찾기
2. `<hp:t>` 태그 → 텍스트, `<hp:script>` 태그 → `[수식]` 형태
3. 미주(`<hp:endNote>`) 자동 감지 — 해설이 미주에 저장된 형식 지원

#### `split_problems(text) → list`

추출된 텍스트를 문제별로 분리:
- `1번`, `2번` 등 번호 패턴으로 분리
- 각 항목: `{"number": 1, "text": "...문제+해설..."}`

#### `create_hwpx(result_text, template_bytes=None) → bytes`

Claude 결과 텍스트를 HWPX 파일로 변환:
1. 템플릿 HWPX가 있으면 기반으로 사용, 없으면 새로 생성
2. `-문제-`, `-해설-`, `-유사문항-` 등 태그로 본문 구성
3. `[수식코드]` → `<hp:equation>...<hp:script>수식</hp:script>...</hp:equation>` 변환
4. XML-escape 처리 (특수문자 보호)
5. ZIP으로 패킹하여 `.hwpx` 바이트 반환

### GraphService (`graph_service.py`)

Claude 출력 내 `-그래프-` 태그를 Matplotlib PNG로 변환.

#### `process_graphs_in_text(text) → (str, list)`

텍스트에서 그래프 태그를 찾아 처리:
- 입력: `-그래프-\n함수: y=x^2-4x+3\nx범위: -1,5\n...\n-그래프끝-`
- 출력: 태그가 제거된 텍스트 + `graphs` 리스트

#### 지원 태그 키

| 키 | 설명 |
|----|------|
| `함수:` | 그릴 함수식 (예: `y=x^2-4x+3`) |
| `x범위:` | x축 범위 (예: `-1,5`) |
| `y범위:` | y축 범위 (예: `-2,4`) |
| `포인트:` | 특정 점 표시 (예: `(2,-1),(1,0)`) |
| `라벨:` | 점 라벨 (예: `A,B`) |
| `점선:` | 점선 보조선 좌표 |
| `점근선:` | 점근선 |
| `음영:` | 음영 영역 |
| `직선:` | 직선 (기울기, y절편) |
| `원:` | 원 (중심x, 중심y, 반지름) |
| `격자:` | 격자 표시 여부 |

그래프 스타일은 `backend/data/graph_style_profile.json` 기반 (수능/교과서 스타일).

## 9. 수식 처리 파이프라인

### 입력 → 출력 변환

```
[원본 수학 문제]
      ↓ (Claude 처리)
[한글 수식 코드] → [수식코드] 형태
      ↓ (프론트엔드 SolutionDisplay.jsx)
```

### HWP 수식 코드 규칙 (핵심)

| 수학 표기 | HWP 코드 |
|---------|---------|
| ± | `+-` |
| ∞ | `inf` |
| ∴ | `therefore` |
| → | `->` |
| 분수 a/b | `{a} over {b}` |
| 루트 √a | `sqrt{a}` |
| 위첨자 | `a^{n}` |
| 아래첨자 | `a_{n}` |
| 적분 | `int_{a}^{b}` |
| 극한 | `lim_{x -> a}` |
| 경우의 수 | `cases{ 값1 && 조건1 # 값2 && 조건2 }` |
| 로마체 | `rm{sin}`, `rm{cos}` |

### HWPX 수식 XML 변환

`create_hwpx()` 내부에서 `[수식코드]` → XML:

```xml
<hp:equation>
  <hp:equationCaption/>
  <hp:script>수식코드</hp:script>
  <hp:para>
    <hp:run>
      <hp:runPr/>
      <hp:t>수식코드</hp:t>
    </hp:run>
  </hp:para>
</hp:equation>
```

## 11. HWPX 처리 파이프라인

### 읽기 파이프라인

```
HWPX 파일 (ZIP)
      ↓
Contents/section0.xml 추출
      ↓
<hp:t> 태그 → 텍스트
<hp:script> 태그 → [수식코드]
<hp:endNote> → 미주(해설) 자동 인식
      ↓
문제별 분리 (1번, 2번 패턴)
      ↓
{"number": 1, "text": "문제+해설"} 리스트
```

### 쓰기 파이프라인

```
Claude 결과 텍스트 (-문제-, -해설- 태그)
      ↓
태그별 파싱
      ↓
[수식코드] → <hp:equation> XML 변환
      ↓
section XML 생성
      ↓
ZIP 패킹
      ↓
HWPX 바이트 → 메모리 저장 (download_id 발급)
      ↓
GET /api/hwpx-download/{id} 로 다운로드
```

### 임시 파일 관리

생성된 HWPX는 메모리(`_hwpx_store`)에 저장:
- 최대 보관 시간: 1시간
- `download_id`: UUID 앞 8자리
- 다운로드 후 즉시 메모리에서 삭제 (`.pop()`)
- 1시간 초과 항목은 다음 요청 시 자동 정리 (`_cleanup_store()`)

## 12. 스캔 워크플로우

### 상세 처리 흐름

```
1. 파일 업로드 (이미지/PDF)
   ├── PDF → PyMuPDF로 페이지별 PNG 변환 (DPI 150)
   └── 이미지 → base64 인코딩

2. Gemini bbox 감지 (detect_problem_bboxes)
   ├── Gemini에게 문제 영역 위치 요청
   └── 비율 좌표(0~1) 형태로 반환

3. 사용자 bbox 확인/수정
   ├── 박스 삭제 (실수로 감지된 것)
   ├── 박스 추가 (누락된 문제)
   └── output_mode 선택

4. SSE 스트리밍 처리 (/api/scan-crop-process)
   문제별 순서대로:
   a. bbox 좌표로 이미지 크롭 (Pillow)
   b. 크롭된 이미지를 Gemini OCR
      → {"problem": "...", "solution": "...", "has_solution": bool}
   c. Claude로 HWP 변환 + 해설/유사문항 처리
   d. 결과 SSE 전송 → 프론트엔드 실시간 표시

5. 전체 완료 후 HWPX 다운로드 가능
```

### 이미지 크롭 (`_crop_image`)

```python
def _crop_image(image_base64, media_type, x, y, w, h):

    right = int((x + w) * iw)
    bottom = int((y + h) * ih)

    cropped = img.crop((left, top, right, bottom))
```

## 14. 지침(Guidelines) 시스템

### 목적

반복적으로 사용하는 추가 지시사항(custom_prompt)을 저장하고 재사용.

### 저장 위치

브라우저 `localStorage` → `mathsol_guidelines` 키

### 데이터 구조

```json
[
  {
    "id": 1713400000000,
    "name": "수능 스타일",
    "content": "수능 형식으로 작성하고 4지선다 형태로 만들어주세요."
  }
]
```

### 사용 흐름

1. 상단 "지침" 버튼 클릭 → `GuidelinesModal` 다이얼로그 열림
2. 지침 작성 → 이름 입력 → 저장
3. 저장된 지침 목록에서 "적용" 클릭 → `guidelines` 상태 업데이트
4. 탭 컴포넌트는 `guidelines`를 `custom_prompt`로 API에 전달

### 다크/라이트 모드

- `localStorage` → `ms_theme` 키에 `'dark'` 또는 `'light'` 저장
- 기본값: 다크 모드
- `document.documentElement.classList.toggle('dark', dark)` 로 Tailwind dark 모드 토글

## 15. 비용 계산 시스템

### 계산 공식

```python
PRICING = {
    "claude-sonnet-4-20250514": {"input": 3, "output": 15},    # USD per 1M tokens
    "claude-opus-4-20250514": {"input": 15, "output": 75},
}

cost_usd = (input_tokens * input_price + output_tokens * output_price) / 1_000_000
cost_krw = cost_usd * 1450  # 대략적 환율
```

### 2차 호출 포함 비용

유사문항 생성 등은 2회 Claude 호출이므로 `_merge_usage()`로 합산:

```python
total_usage = {
    "input_tokens": usage1["input_tokens"] + usage2["input_tokens"],
    "output_tokens": usage1["output_tokens"] + usage2["output_tokens"],
    "total_tokens": ...,
    "cost_usd": usage1["cost_usd"] + usage2["cost_usd"],
    "cost_krw": ...,
    "model": usage1["model"],
}
```

### 비용 표시 (UsageInfo 컴포넌트)

- 총 토큰 수, 모델명, 비용(USD + KRW) 표시
- 일괄 처리는 전체 합산 비용 표시
